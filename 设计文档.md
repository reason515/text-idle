# 游戏名称
Text Idle

# 高阶理念
一款深度策略型文字放置RPG。游戏致力于将《魔兽世界》的职业战术、《暗黑破坏神》的装备驱动、《流放之路》的物品制作等经典设计精髓，提炼并浓缩于纯粹的文字体验之中。我们摒弃繁杂的图形，旨在打造一个让玩家能心无旁骛、深度钻研策略与构筑（Build）的硬核游戏环境。

# 设计哲学
- **策略优先**: 游戏的核心乐趣源于深度的策略思考，而非频繁的操作。通过离线挂机机制解放玩家的时间，让每一次决策都显得至关重要。
- **透明化设计**: 采用"小数值"系统，所有属性、伤害计算、怪物行为逻辑等底层公式完全对玩家开放。我们鼓励玩家通过数据分析、战斗日志复盘等方式，成为精通游戏机制的大师。
- **致敬经典**: 汲取ARPG与MMORPG黄金时代的灵感，重现"战法牧"铁三角、仇恨机制、惊喜掉落以及装备词缀系统带来的无穷乐趣。

# 核心游戏循环
- **组建 (Assemble)**: 招募并组建一支五人小队。根据职业特性（如坦克、治疗、伤害输出），打造一支独一无二的冒险队伍。
-  **布局 (Strategize)**: 为每位角色精细配置战斗AI。设定技能释放的优先级、触发条件以及团队战术，让你的策略自动执行。
- **放置 (Engage)**: 队伍将依据你的策略自动进行探索与战斗，获取经验、金币与装备。即使玩家离线，探索也从不间断。
- **分析 (Analyze)**: 通过实时的战斗数据面板与详尽的挂机收益统计，精准评估当前策略的效率与短板。
- **优化 (Optimize)**: 使用获取的资源与装备强化你的队伍。调整技能搭配、更新装备、优化战术，以挑战更强大的敌人，追求更高的挂机收益。

# 关键特征
- **深度策略小队**: 经典的"坦克、治疗、DPS"角色定位与仇恨机制，要求玩家深刻理解职业 synergy，构筑平衡且高效的队伍。
- **装备驱动的乐趣**:
    - **暗黑式词缀系统**: 借鉴《暗黑破坏神2》，设计了从普通到独特的装备分级，以及丰富的随机词缀，让每一次掉落都充满惊喜。
    - **流放式强化系统**: 引入《流放之路》风格的通货道具，玩家可以通过它们来重塑或增强装备词缀，享受制作神装的乐趣。

## 游戏概述
- **核心愿景 (Core Vision)**
	- 致力于为玩家提供一个纯粹、无负担、可长期沉浸的策略型放置游戏体验。我们旨在让玩家能够轻松利用碎片时间进行深度策略研究与享受游戏乐趣，而非被繁琐操作所束缚。
- **目标用户 (Target Audience)**
    - 追求深度策略与机制研究，乐于进行理论构筑（Theorycrafting）的核心玩家。
    - 热衷于《魔兽世界》、《暗黑破坏神》等经典RPG，享受团队配合与装备驱动乐趣的玩家群体。
    - 忙碌的上班族或学生党，渴望利用碎片时间放松，避免高频操作负担，但仍能体验到游戏深度。
- **独特卖点 (Unique Selling Points)**
    - **极致透明的机制**: 游戏所有底层机制、属性、计算公式完全公开，鼓励玩家深度挖掘，成为真正的策略大师。
    - **自由定制的战术**: 提供高度灵活的作战策略配置，玩家可自由定义队伍AI，实现千变万化的战术组合。
    - **数据驱动的优化**: 详尽的战斗数据统计与分析功能，帮助玩家直观评估策略效果，形成"策略制定-数据验证-迭代优化"的正向循环。

# 核心玩法与循环
- **组建 (Assemble)**: 角色获取与队伍构成
	- 队伍规模随进度逐步解锁：从 1 名角色起步，每推进一张地图增加一个席位，最终最多 5 人，让玩家在职业与机制上循序渐进，避免初期信息过载。
	- 每名新成员加入时，由玩家为其选择职业与初始技能，从而塑造独一无二的队伍构成。
- **布局 (Strategize)**: 战斗策略与 AI 配置
	- 在"坦克、治疗、DPS"定位与仇恨机制的基础上，为每位角色配置战斗 AI：自定义技能（含普攻）的释放条件、目标选择与优先级顺序，使策略自动执行。
	- 策略是否合理需结合战斗日志与数据统计来验证与迭代，形成"配置—观察—调整"的闭环。
- **放置 (Engage)**: 关卡、怪物与离线收益
	- 队伍按当前策略自动探索与战斗，持续获取经验、金币与装备；支持离线挂机，探索不中断。
	- 离线期间按真实战斗逻辑结算，数据统计与装备掉落与在线时一致，保证收益可预期。
	- 离线挂机设有最大累积时长，超时未上线则不再累积收益，鼓励定期回访。
	- 单场战斗的怪物数量多数与队伍人数一致，少数场次会多于或少于，用于调节难度与节奏。
	- 关卡与怪物命名借鉴《魔兽世界》经典地图与生物，强化熟悉感与代入感。
	- 怪物分为普通、精英、首领三档，强度与掉落品质逐级提升。
	- 通过击杀怪物积累关卡进度，达到要求后解锁后续关卡，驱动推进节奏。
	- 关卡探索中可能"偶遇"其他也在当前关卡挂机的玩家，触发 PVP；战斗结果单独统计，战胜可获得荣誉积分，用于兑换稀有道具，并参与荣誉排行榜。
- **分析 (Analyze)**: 战斗日志与数据统计
	- 通过实时战斗面板与挂机收益统计，评估策略效率、定位短板，为优化提供依据。
- **优化 (Optimize)**: 角色成长与装备驱动
	- 用获得的资源与装备强化队伍：升级技能、更换词缀装备、调整战术，以挑战更强敌人、提升挂机收益。

# 关卡、地图与怪物设计详细说明

## 一、地图系统 (Map System)

### 1.1 解锁机制（大秘境式探索）

借鉴《暗黑破坏神》大秘境的设计思路，地图采用**探索度驱动**的推进模式：

| 阶段 | 机制说明 |
|------|----------|
| **初始状态** | 玩家仅能选择**初始地图**进行探索，无法进入后续地图 |
| **刷怪积累** | 在地图上持续战斗、击杀怪物，逐步积累**探索度 (Exploration Progress)** |
| **BOSS 出现** | 探索度达到 100% 时，该地图的**守关 BOSS** 出现 |
| **解锁下一张** | 击败 BOSS 后，解锁下一张地图，并可选择新地图继续推进 |

- **探索度公式**：每击杀一只怪物，根据怪物等级与类型贡献一定探索度；普通怪贡献较少，精英怪贡献较多，BOSS 不计入探索度（为守关目标）。
- **节奏控制**：探索度增长速度可调，用于控制单张地图的游玩时长与推进节奏。

### 1.2 英雄席位与地图解锁绑定

队伍规模与地图进度严格绑定，形成**循序渐进**的成长曲线：

| 地图序号 | 队伍席位 | 说明 |
|----------|----------|------|
| 初始 | 1 人 | 玩家仅能选择 1 位英雄加入队伍 |
| 解锁第 2 张地图 | 2 人 | 每解锁一张新地图，可再招募 1 位英雄 |
| 解锁第 3 张地图 | 3 人 | 同上 |
| 解锁第 4 张地图 | 4 人 | 同上 |
| 解锁第 5 张地图 | 5 人 | 队伍满编，最多 5 人 |

- **设计意图**：避免初期信息过载，让玩家在熟悉单英雄操作后再逐步扩展队伍，同时强化「解锁新地图」的奖励感。

### 1.3 经典地图列表（魔兽世界风格）

采用《魔兽世界》中玩家耳熟能详的经典地图及对应怪物，强化熟悉感与代入感。地图按难度递进排列：

| 序号 | 地图英文名 | 地图中文名 | 典型怪物（示例） | 守关 BOSS |
|------|------------|------------|------------------|-----------|
| 1 | Elwynn Forest | 艾尔文森林 | Kobold, Wolf, Defias | Hogger |
| 2 | Westfall | 西部荒野 | Defias, Harvest Golem, Vulture | VanCleef |
| 3 | Duskwood | 暮色森林 | Skeleton, Worgen, Zombie | Stitches |
| 4 | Redridge Mountains | 赤脊山 | Blackrock Orc, Spider | Kazon |
| 5 | Stranglethorn Vale | 荆棘谷 | Troll, Tiger, Panther | King Bangalash |

- **命名规范**：地图与怪物均采用 WoW 经典译名，便于玩家识别。
- **扩展性**：后续版本可追加更多地图（如 Burning Steppes、Un'Goro 等），保持相同解锁逻辑。

## 二、怪物系统 (Monster System)

### 2.1 怪物等级划分

怪物分为三个等级，强度与机制复杂度逐级提升：

| 等级 | 英文标识 | 强度特点 | 技能使用 | 掉落品质 |
|------|----------|----------|----------|----------|
| 普通 | Normal | 基础属性，无特殊机制 | 仅普攻 | 普通/魔法 |
| 精英 | Elite | 生命/伤害显著提升 | **会使用技能**，需谨慎应对 | 魔法/稀有 |
| 首领 | Boss | 最高强度，多阶段或特殊机制 | 多技能组合，高威胁 | 稀有/独特 |

- **精英及以上**：从精英开始，怪物会使用技能（如打断、AOE、控制等），玩家需根据仇恨、治疗与打断节奏调整策略。
- **Boss**：守关 BOSS 拥有更强技能与机制，是检验队伍构筑与策略的关键节点。

### 2.2 伤害类型

怪物可造成**物理伤害**或**魔法伤害**，与玩家的护甲/抗性体系对应：

| 伤害类型 | 英文标识 | 减伤属性 | 典型怪物示例 |
|----------|----------|----------|--------------|
| 物理 | Physical | 护甲 (Armor) | Wolf, Defias, Orc, Tiger |
| 魔法 | Magic | 抗性 (Resistance) | Mage-type Kobold, Shadow Wolf, Caster Troll |

- **混合型怪物**：部分精英或 BOSS 可同时拥有物理与魔法攻击，考验玩家的双抗配置。
- **透明化**：怪物面板与战斗日志中明确标注伤害类型，便于玩家针对性堆叠护甲或抗性。

### 2.3 怪物属性设计 (Monster Attributes)

怪物采用**简化属性体系**，与英雄共用同一套伤害/减伤公式，保证战斗计算透明且易于配置。

#### 2.3.1 核心战斗属性

| 属性 | 英文标识 | 用途 | 说明 |
|------|----------|------|------|
| 生命值 | HP | 承受伤害，归零即死亡 | 公式见下方，受类型系数与等级系数影响 |
| 物理攻击力 | PhysAtk | 造成物理伤害 | 普攻与物理技能的基础数值 |
| 法术强度 | SpellPower | 造成魔法伤害 | 法术技能的基础数值 |
| 敏捷 | Agility | 出手顺序 | 与英雄共用「敏捷高者先手」规则；同敏捷随机 |
| 护甲 | Armor | 减免所受物理伤害 | 英雄攻击怪物时，怪物护甲参与减伤计算 |
| 法术抗性 | Resistance | 减免所受魔法伤害 | 英雄攻击怪物时，怪物抗性参与减伤计算 |

- **防御公式**：与英雄一致，1 护甲 = 抵消 1 点物理伤害，1 抗性 = 抵消 1 点魔法伤害，无上限。
- **伤害公式**：怪物对英雄造成伤害时，使用英雄的护甲/抗性；英雄对怪物造成伤害时，使用怪物的护甲/抗性。

#### 2.3.2 可选次级属性（精英/BOSS）

| 属性 | 英文标识 | 用途 | 说明 |
|------|----------|------|------|
| 物理暴击率 | PhysCrit | 物理攻击暴击概率 | 普通怪可设为 0 或 5% 基础值 |
| 法术暴击率 | SpellCrit | 法术攻击暴击概率 | 同上 |
| 闪避率 | Dodge | 躲避物理攻击的概率 | 高敏捷型怪物可适当提高 |
| 命中率 | Hit | 攻击命中概率 | 多数怪物可固定为 95% 基础，BOSS 可略高 |

- **简化原则**：普通怪可仅配置核心属性，次级属性使用默认值；精英与 BOSS 按需配置，体现差异化。

#### 2.3.3 怪物技能 (Monster Skills)

精英与 BOSS 拥有**技能**，普攻与技能伤害倍率不同（技能约 1.25x）：

| 技能 ID | 英文名 | 倍率 | 冷却 | 效果描述 | 适用怪物 |
|---------|--------|------|------|----------|----------|
| stone-shard | Stone Shard | 1.2x | 2 回合 | 1.2x 魔法 + **Splinter**（抗性 -2） | Kobold Geomancer |
| blackjack | Blackjack | 1.35x | 2 回合 | 纯爆发伤害 | Defias Smuggler |
| swift-cut | Swift Cut | 1.1x | 2 回合 | 1.1x 物理 + **Bleed**（DOT） | Defias Cutpurse |
| rend | Rend | 1.5x | 3 回合 | 纯爆发伤害 | Hogger |

- **冷却**：使用技能后需等待 N 回合才能再次使用；BOSS 技能冷却略长。
- **特色**：Stone Shard 偏 debuff；Blackjack 纯爆发；Swift Cut 偏 DOT；Rend 纯高伤。
- **DOT 结算**：每回合结束时，Bleed 对目标造成伤害/回合，持续剩余回合数；回合结束时 tick 减 1。

- **技能概率**：精英 35%，BOSS 55%；普通怪无技能（skillChance=0）。
- **怪物详情**：点击怪物卡片可查看其技能名称与效果描述。

#### 2.3.4 伤害类型与属性对应

| 怪物类型 | PhysAtk | SpellPower | PhysRatio | MagicRatio | 示例 |
|----------|---------|------------|-----------|------------|------|
| 纯物理 | 有值 | 0 | 1.0 | 0 | Wolf, Defias, Orc |
| 纯魔法 | 0 | 有值 | 0 | 1.0 | Kobold Geomancer, Caster Troll |
| 混合型 | 有值 | 有值 | 0.5 | 0.5 | 部分精英、BOSS |

- 普攻与技能根据 PhysRatio/MagicRatio 选择使用 PhysAtk 或 SpellPower 参与伤害计算。

#### 2.3.5 属性公式与配置参数

```
HP         = Base_HP * TierMult * (1 + Level * LevelScale)
PhysAtk    = Base_PhysAtk * TierMult * (1 + Level * LevelScale)
SpellPower = Base_SpellPower * TierMult * (1 + Level * LevelScale)
Agility    = Base_Agility * TierMult * (1 + Level * LevelScale)
Armor      = Base_Armor * TierMult * (1 + Level * LevelScale) + floor(Level * 0.5)
Resistance = Base_Resistance * TierMult * (1 + Level * LevelScale) + floor(Level * 0.5)
```

- **Armor/Resistance 等级底数**：即使 Base 为 0，高等级怪物也会获得 `floor(Level * 0.5)` 的护甲/抗性，保证随等级合理成长。

| 参数 | 说明 | 示例 |
|------|------|------|
| Base_* | 每类怪物的基础值 | 如 Young Wolf: Base_HP=30, Base_PhysAtk=8 |
| TierMult | 类型系数：Normal≈1.15, Elite≈1.5, Boss≈4（普通怪略强、精英略弱，缩小差距） | 可配置 |
| Level | 怪物等级（与队伍等级挂钩，单张地图内怪物等级在 [baseLevel+min, baseLevel+max] 范围内随机） | 1–60 |
| LevelScale_* | 等级缩放系数 | 如 LevelScale=0.16（约每级 14% 提升，与玩家每级 5 属性点强度相当） |

- **小数值原则**：1 级普通怪 HP 约 20–50，PhysAtk/SpellPower 约 5–15，与 1 级英雄量级相当；随等级与类型系数放大。

- **等级范围**：每张地图可配置 `levelRange: { min, max }`，表示相对 baseLevel 的偏移。例如 `{ min: -1, max: 2 }` 表示怪物等级在 [baseLevel-1, baseLevel+2] 内随机。

## 三、与核心循环的衔接

- **放置 (Engage)**：队伍在选定地图上自动刷怪，探索度持续累积；探索度满后 BOSS 出现，击败即解锁下一张地图。
- **组建 (Assemble)**：每解锁一张新地图，可招募一名新英雄，直至队伍满 5 人。
- **优化 (Optimize)**：根据怪物伤害类型（物理/魔法）调整装备与抗性；根据精英/BOSS 技能调整战术与打断优先级。

# 战斗系统设计详细说明

## 一、回合与出手顺序 (Turn Order)

### 1.1 出手顺序规则

战斗采用**回合制**，每回合内所有参战单位（玩家英雄与怪物）按**敏捷 (Agility)** 从高到低依次行动：

| 规则 | 说明 |
|------|------|
| **敏捷决定顺序** | 敏捷越高的角色（含怪物）越先出手 |
| **同敏捷处理** | 敏捷相同时，出手顺序随机决定 |
| **透明化** | 战斗日志中记录出手顺序与行动结果，便于战后复盘与战术调整（本游戏为放置游戏，战斗过程玩家无法实时操控，仅能通过预先配置的战术策略间接影响） |

- **设计意图**：敏捷不仅影响暴击、闪避，还直接影响行动先后，使高敏捷职业（盗贼、猎人、德鲁伊）在节奏上有优势。

### 1.2 每回合行动次数

| 规则 | 说明 |
|------|------|
| **每回合每角色行动一次** | 每个参战单位（英雄或怪物）每回合仅能执行一次行动（普攻或技能） |
| **行动类型** | 行动可为普攻、主动技能或待机等，由战斗 AI 或玩家策略决定 |

- **小数值与节奏**：单回合单行动保证战斗节奏清晰，便于仇恨、治疗与打断的时序规划。

---

## 二、怪物数量与战斗规模 (Encounter Size)

### 2.1 怪物数量分布

单场战斗中怪物数量与玩家小队规模的关系如下：

| 情况 | 概率 | 说明 |
|------|------|------|
| **与队伍人数相当** | 较大概率 | 如 3 人队伍时，多数场次为 3 只怪 |
| **少于队伍人数** | 较小概率 | 如 3 人队伍时，少数场次为 1–2 只怪，节奏更快 |
| **多于队伍人数** | 较小概率 | 如 3 人队伍时，少数场次为 4–5 只怪，难度提升 |

- **可配置**：具体概率分布（如 70% 相等、15% 少、15% 多）可通过配置调节，便于平衡调试。
- **设计意图**：多数场次势均力敌，少数场次制造节奏变化与难度起伏。

## 三、胜负判定 (Victory & Defeat)

| 结果 | 触发条件 | 后续 |
|------|----------|------|
| **胜利** | 怪物全部死亡 | 进入战后结算（经验、金币、掉落）；进入休息恢复阶段（自动执行） |
| **失败** | 玩家英雄全部死亡 | 战斗结束；不获得本场掉落；**扣除当前地图探索度 10 点（可配置），最低不低于 0**；短暂等待后自动开始下一场战斗 |

- **战败惩罚**：每次战败扣除探索度（默认 10 点，可配置），防止零成本无限刷怪；探索度不会因战败归零，保留已积累的进度。
- **透明化**：胜负判定逻辑对玩家完全公开，便于评估风险与策略取舍。

---

## 四、战后休息与恢复 (Post-Combat Rest)

### 4.1 恢复机制

每场战斗胜利后，队伍自动进入**休息阶段**，无需玩家手动操作，系统逐步恢复所有角色资源，直至完全恢复后自动开始下一场战斗：

| 规则 | 说明 |
|------|------|
| **自动执行** | 休息恢复全程自动进行，玩家无需点击任何按钮 |
| **恢复门槛** | 所有英雄 HP 与 MP（或怒气/能量/集中值等）均恢复至满值后，自动开始下一场战斗 |
| **恢复速度** | HP、MP 的恢复速度受**精神 (Spirit)** 属性以及**装备**等因素影响 |
| **死亡惩罚** | 若有角色在本场战斗中死亡，恢复时间将延长（如恢复速度降低或需额外恢复回合），以体现战斗代价 |
| **时间单位** | 恢复按**回合数**或**时间步**结算（遵循游戏「回合制」设计，不使用秒/分钟等实时单位） |

### 4.2 恢复公式（草案）

| 资源 | 恢复公式（每回合/每步） | 主要影响因素 |
|------|--------------------------|--------------|
| HP | `HP_Regen = Base + Spi * k_HP_Regen + 装备恢复加成` | 精神、装备词缀 |
| MP | `MP_Regen = Base + Spi * k_MP_Regen + 装备恢复加成` | 精神、装备词缀 |

- **怒气/能量/集中值**：战士怒气脱战后归零，休息阶段不恢复；其他资源按职业规则处理。
- **死亡惩罚**：若有 N 名角色在本场战斗中死亡，恢复速度可乘以惩罚系数（如 `1 + N * k_DeathPenalty`），或增加固定额外恢复回合，具体数值可配置调节。
- **透明化**：恢复速度与公式对玩家可见，便于评估「精神」与恢复词缀的收益。

### 4.3 与核心循环的衔接

- **放置 (Engage)**：战斗 → 休息恢复 → 下一场战斗，形成连续的探索节奏。
- **优化 (Optimize)**：堆叠精神或恢复词缀可缩短休息时间，提高挂机效率。

- **仇恨机制**：怪物每回合按仇恨值选择目标，详见「仇恨机制」章节。
- **伤害与减伤**：物理/魔法伤害、护甲/抗性减伤公式见「职业与属性设计」2.2.3。
- **状态效果**：Buff/Debuff 对出手顺序、恢复等的影响，见「状态效果」设计。

# 职业与属性设计详细说明

## 一、职业设计 (Classes)

### 1.1 定位与职责

| 定位 | 英文标识 | 职责简述 | 核心能力 |
|------|----------|----------|----------|

### 1.4 主属性与成长

- 每个职业有 **主属性 (Primary Attribute)**，升级与装备提供的该属性收益更高。
- 职业可拥有 **副定位**（如圣骑士可 Tank/Heal/DPS），通过技能与装备选择实现不同玩法。
- **升级自由分配**：每次升级获得 **5 点主属性**，由玩家自由分配至力量、敏捷、智力、耐力、精神中的任意一项。满级（60 级）时累计可获得 `5 × (60 - 1) = 295` 点自由属性。

### 1.5 职业与技能关系（详见「技能系统设计详细说明」）

- 职业决定可学习的 **技能池**；玩家从池中选择技能装备到角色。
- 部分技能为职业专属，部分为通用（如普攻变体）。
- 技能获取方式：初始技能选择、等级解锁（5 的倍数）、装备/道具解锁（后续扩展）。

### 1.6 经验与升级 (Experience & Leveling)

#### 1.6.1 经验获取

| 规则 | 说明 |
|------|------|
| **获取时机** | 仅在**战斗胜利**后获得经验；战败不获得经验 |
| **分配对象** | 参与战斗的英雄**平均分配**本场战斗获得的经验，每位英雄获得相同经验值 |
| **经验来源** | 击杀怪物贡献经验；怪物等级、类型（普通/精英/首领）影响单次经验量 |

- **设计意图**：胜利导向，鼓励玩家优化策略与构筑以稳定取胜；战败无经验避免「送死刷经验」等不合理行为。

#### 1.6.2 升级条件与奖励

| 规则 | 说明 |
|------|------|
| **升级条件** | 角色经验累积达到当前等级所需经验值时自动升级 |
| **升级奖励** | 每次升级获得 **5 点可自由分配的主属性**，由玩家分配至力量、敏捷、智力、耐力、精神中的任意一项 |
| **满级** | 60 级为满级；满级后经验不再累积，自由属性累计 `5 × (60 - 1) = 295` 点 |

- **透明化**：经验条、升级所需经验值对玩家完全可见，便于规划成长节奏。

#### 1.6.3 经验曲线与节奏控制

升级节奏采用**较慢曲线**设计，让玩家有充足时间熟悉游戏机制、研究构筑与战术：

| 设计原则 | 说明 |
|----------|------|
| **节奏放慢** | 每级所需经验随等级递增，中后期升级间隔明显拉长 |
| **熟悉期** | 前期（1–10 级）升级相对较快，便于玩家建立基础认知 |
| **深耕期** | 中后期升级放缓，鼓励玩家在单等级内充分探索装备、技能与战术搭配 |

**经验曲线公式（草案）**：

```
XP_Required(Level) = Base_XP * (Level ^ Curve_Exponent)
```

| 参数 | 说明 | 建议值 |
|------|------|--------|
| Base_XP | 1 级升 2 级所需基础经验 | 可配置，如 50 |
| Curve_Exponent | 曲线指数，越大升级越慢 | 建议 1.5–2.0，实现「节奏放慢」 |

- **可配置**：Base_XP、Curve_Exponent 及怪物经验贡献量均可通过配置调节，便于后续平衡迭代。
- **示例**：若 Base_XP=50、Curve_Exponent=1.8，则 1→2 级需 50 XP，10→11 级约需 2000+ XP，30→31 级约需 2 万+ XP，体现明显放缓的节奏。

#### 1.6.4 与核心循环的衔接

- **放置 (Engage)**：战斗胜利 → 获得经验 → 经验累积 → 升级 → 分配属性，形成持续成长闭环。
- **优化 (Optimize)**：升级获得的自由属性点影响构筑方向；较慢节奏促使玩家在每级内更注重装备、技能与战术的优化，而非单纯「刷级」。

## 二、核心属性 (Attributes)

### 2.1 属性定义

采用小数值、透明化设计，所有属性对战斗数值的影响公式对玩家完全公开。

| 属性 | 英文标识 | 主要影响 | 典型受益职业 |
|------|----------|----------|--------------|
| 力量 | Strength | 物理攻击力、护甲、格挡值 | 战士、圣骑士 |
| 敏捷 | Agility | 暴击率、闪避、物理命中 | 盗贼、猎人、德鲁伊 |
| 智力 | Intellect | 法术强度、法力上限、法术暴击 | 法师、牧师、术士、圣骑士 |
| 耐力 | Stamina | 生命值上限 | 全职业（坦克尤其重要） |
| 精神 | Spirit | 法力恢复、生命恢复（脱战） | 治疗职业 |

### 2.2 次级属性与资源系统

#### 2.2.1 资源类型 (Resources)

| 资源 | 英文标识 | 适用职业 | 上限公式 | 恢复方式 |
|------|----------|----------|----------|----------|
| 生命值 | HP | 全职业 | 见 2.2.2 | 治疗技能、脱战精神恢复 |
| 法力值 | MP | 法师、牧师、术士、萨满、圣骑士、德鲁伊 | 见 2.2.2 | 战斗内精神恢复、脱战快速恢复 |
| 怒气 | Rage | 战士 | 固定 100，不随等级变化 | 受到伤害获得、造成伤害获得 |
| 能量 | Energy | 盗贼 | 固定 100，不随等级变化 | 固定速率恢复（如每回合 10 点） |
| 集中值 | Focus | 猎人 | 固定 100 | 固定速率恢复、技能消耗 |

- **小数值原则**：怒气/能量/集中值上限固定为 100，便于技能消耗设计（如 20 怒气、30 能量）。
- **资源差异化**：法力职业依赖精神恢复；怒气职业依赖战斗节奏；能量/集中职业依赖时间与节奏。

#### 2.2.2 次级属性公式（小数值版 + 职业差异化）

所有系数采用小数值，使 1 级角色衍生属性在 10–50 量级，满级（60 级）在 100–350 量级，便于心算与透明化。

**通用公式结构**：`次级属性 = Base + 主属性 * k_职业`，其中 `k_职业` 为职业系数，体现职业对某属性的转化效率差异。

##### 基础公式（职业系数见下表）

| 次级属性 | 英文标识 | 通用公式 | 主属性来源 |
|----------|----------|----------|------------|
| 生命值上限 | HP | `HP = 10 + Stam * k_HP + Level * 2` | 耐力 |
| 法力值上限 | MP | `MP = 5 + Int * k_MP + Level * 1` | 智力（仅法力职业） |
| 物理攻击力 | PhysAtk | `PhysAtk = 5 + 主物理属性 * k_PhysAtk` | 力量或敏捷（见职业表） |
| 法术强度 | SpellPower | `SpellPower = 5 + Int * k_SpellPower` | 智力（仅法术职业） |
| 护甲 | Armor | `Armor = Str * k_Armor + 装备护甲` | 力量 |
| 法术抗性 | Resistance | `Resistance = Int * k_Resistance + 装备抗性` | 智力 |
| 物理暴击率(%) | PhysCrit | `PhysCrit = 5 + Agi * k_PhysCrit` | 敏捷 |
| 法术暴击率(%) | SpellCrit | `SpellCrit = 5 + Int * k_SpellCrit` | 智力 |
| 闪避率(%) | Dodge | `Dodge = 5 + Agi * k_Dodge` | 敏捷 |
| 命中率(%) | Hit | `Hit = 95 + Agi * 0.2` | 敏捷（全职业统一） |
| 格挡率(%) | Block | `Block = 装备/技能提供` | 仅持盾职业 |

##### 职业系数表（k 值）

系数设计考虑满级（60 级）时 295 点自由属性全加主属性的极端情况，使攻击力/法术强度控制在 100–250 量级，符合小数值原则。

| 职业 | k_HP | k_MP | PhysAtk 主属性 | k_PhysAtk | k_SpellPower | k_Armor | k_Resistance | k_PhysCrit | k_SpellCrit | k_Dodge |
|------|------|------|----------------|-----------|--------------|---------|--------------|------------|-------------|---------|
| 战士 | 4.0 | - | Str | 0.65 | - | 0.8 | 0.3 | 0.3 | - | 0.2 |
| 圣骑士 | 3.5 | 2.2 | Str | 0.45 | 0.45 | 0.6 | 0.6 | 0.3 | 0.4 | 0.2 |
| 牧师 | 2.5 | 2.8 | - | - | 0.65 | - | 0.8 | 0.3 | 0.6 | 0.2 |
| 德鲁伊 | 3.2 | 2.2 | Agi | 0.5 | 0.45 | 0.4 | 0.6 | 0.6 | 0.5 | 0.4 |
| 法师 | 2.0 | 2.8 | - | - | 0.65 | - | 0.8 | 0.3 | 0.6 | 0.2 |
| 盗贼 | 2.8 | - | Agi | 0.55 | - | 0.2 | 0.3 | 0.7 | - | 0.5 |
| 猎人 | 3.0 | - | Agi | 0.5 | - | 0.3 | 0.3 | 0.6 | - | 0.4 |
| 术士 | 2.8 | 2.8 | - | - | 0.65 | - | 0.8 | 0.3 | 0.6 | 0.2 |
| 萨满 | 3.0 | 2.2 | Agi | 0.4 | 0.45 | 0.3 | 0.6 | 0.5 | 0.5 | 0.3 |

- **设计说明**：坦克职业（战士）HP/护甲系数高；物理 DPS（盗贼、猎人）暴击/闪避系数高；法术职业（法师、牧师、术士）法术强度/法术暴击系数高；双修职业（圣骑士、德鲁伊、萨满）系数居中，支持多定位。
- **"-" 表示该职业不使用该次级属性**，公式不参与计算。

##### 满级属性估算（60 级，含 295 点自由分配）

假设玩家将全部 295 点自由属性投入主属性（极端主属性构筑），则 60 级时主属性 ≈ 1级基础 + 295。示例（战士，全加力量）：

| 职业 | 1级主属性 | 60级主属性（全加主属性） | HP 估算 | PhysAtk/SpellPower 估算 |
|------|-----------|--------------------------|---------|-------------------------|
| 战士 | Str 10 | 10 + 295 = 305 | 10 + 9×4 + 60×2 = 166 | 5 + 305×0.65 ≈ 203 |
| 法师 | Int 11 | 11 + 295 = 306 | 10 + 4×2 + 60×2 = 138 | 5 + 306×0.65 ≈ 204 |
| 盗贼 | Agi 11 | 11 + 295 = 306 | 10 + 6×2.8 + 60×2 ≈ 147 | 5 + 306×0.55 ≈ 173 |

- **均衡分配**：若玩家将 295 点分散至多属性（如坦克加耐力、DPS 加主属性），实际数值介于 1 级与上述极端值之间。装备词缀会进一步放大差异。

#### 2.2.3 伤害与减伤公式

| 类型 | 公式 | 说明 |
|------|------|------|
| 原始伤害 | `rawDamage = PhysAtk/SpellPower * SkillCoeff * [1.5 if crit]` | SkillCoeff 为技能倍率，暴击时乘 1.5 |
| 护甲抵消 | `finalDamage = max(1, rawDamage - Armor)` | 1 护甲 = 抵消 1 点物理伤害，无上限，装备可叠加 |
| 抗性抵消 | `finalDamage = max(1, rawDamage - Resistance)` | 1 抗性 = 抵消 1 点魔法伤害 |
| 暴击伤害 | `CritMultiplier = 1.5` | 暴击时伤害 * 1.5，可由装备/技能修改 |

- **直接抵消**：每点护甲/抗性 = 抵消 1 点伤害，无上限，装备可叠加。

#### 2.2.4 怒气与能量规则（草案）

| 资源 | 获得方式 | 消耗示例 |
|------|----------|----------|
| 怒气 | 受到伤害: +1 怒气/2 点伤害（至少 1 点）；造成伤害: +1 怒气/4 点伤害 | 英勇打击 15 怒气，盾牌猛击 20 怒气 |
| 能量 | 固定每回合恢复 10 点 | 影袭 40 能量，背刺 60 能量 |
| 集中值 | 固定每回合恢复 6 点；稳固射击不消耗 | 奥术射击 30 集中值 |

- 具体数值在技能设计时平衡，此处为量级参考。

### 2.3 属性来源

| 来源 | 说明 |
|------|------|
| 基础属性 | 职业与等级决定的基础值 |
| 升级成长 | 每级获得固定点数，按职业主属性分配 |
| 装备词缀 | 装备提供 +Str、+Agi、+Int、+护甲、+法术抗性 等词缀 |
| Buff/Debuff | 技能或状态效果临时增减属性 |

### 2.4 属性显示与透明化

- 角色面板展示：**基础属性** + **装备加成** + **Buff 加成** = **最终属性**。
- 战斗日志与数据统计中，可展示关键衍生数值（如实际暴击率、命中率），便于玩家验证与优化构筑。
- **Buff/Debuff 面板提示**：角色或怪物身上存在 buff/debuff 时，在其面板上显示相应徽章；鼠标悬停可查看详细说明（名称、剩余回合、效果描述）。

# 技能系统设计详细说明

## 一、设计思路概述

技能系统从《魔兽世界》经典职业技能中汲取灵感，经适配与改造后服务于回合制战斗。核心设计围绕**技能来源与适配**、**初始选择**、**成长节奏**、**释放控制**四个维度展开。

## 二、技能来源与适配

### 2.1 选型原则

| 原则 | 说明 |
|------|------|
| **来源** | 从《魔兽世界》经典职业技能中选择适合回合制游戏的技能 |
| **适配** | 对不适合的技能进行改造：删除实时单位（秒、分钟），改为回合制单位（回合、每回合） |
| **职业绑定** | 每个职业拥有独立的技能池，技能为职业专属 |

### 2.2 适配示例

| 原 WoW 设计 | 改造后（回合制） |
|------------|------------------|
| 3 秒冷却 | 3 回合冷却 |
| 每秒恢复 10 法力 | 每回合恢复 10 法力 |
| 持续 12 秒的 Buff | 持续 12 回合的 Buff |

- **设计意图**：保持 WoW 技能风味，同时严格遵循「回合制」设计哲学（见 game-design.mdc）。

## 三、初始技能选择

### 3.1 触发时机

| 规则 | 说明 |
|------|------|
| **触发** | 新角色加入队伍时（招募时） |
| **选择数量** | 从本职业的 **3 个初始技能** 中选择 **1 个** |
| **结果** | 所选技能成为该角色首个可用技能，可立即在战斗中使用 |

### 3.2 设计意图

- 让玩家在招募阶段即参与构筑决策，塑造角色定位（如坦克选仇恨技能、治疗选治疗技能）。
- 3 选 1 提供差异化，同职业不同角色可走不同路线。

## 四、技能成长（等级解锁）

### 4.1 触发等级

| 规则 | 说明 |
|------|------|
| **触发等级** | 角色等级为 **5 的倍数** 时（5、10、15、20、25…60） |
| **选择窗口** | 弹出技能选择界面，玩家可选择加强或学习新技能；若暂不选择，游戏继续进行（与升级属性加点类似） |

### 4.2 二选一机制

每次触发时，玩家在以下两种选项中**二选一**：

| 选项 | 说明 |
|------|------|
| **加强已有技能** | 提升该技能的数值或效果（如伤害倍率、治疗量、冷却缩短等） |
| **学习新技能** | 从技能池中学习一个尚未拥有的新技能 |

### 4.3 学习新技能时的可选池

| 规则 | 说明 |
|------|------|
| **可选数量** | 每个选择窗口提供 **3 个固定的可选新技能** |
| **三系分布** | 3 个选项分别来自职业的三个派系（如战士：武器、狂暴、防护），便于玩家按派系构筑 |
| **固定性** | 同一等级、同一职业的 3 个选项为固定组合（非随机），便于玩家规划与攻略 |
| **排除已学** | 已拥有的技能不会出现在可选列表中 |
| **池耗尽** | 若技能池中未学技能不足 3 个，则显示剩余全部；若已学满，则仅显示「加强已有技能」 |

### 4.4 成长节奏示例

| 等级 | 可选操作 |
|------|----------|
| 5 | 加强已有技能，或学习技能 A/B/C（3 选 1，分属三系） |
| 10 | 加强已有技能，或学习技能 D/E/F（3 选 1，分属三系） |
| … | … |
| 60 | 加强已有技能，或学习高级技能（3 选 1，分属三系） |

- **满级技能数**：60 级时共触发 12 次（5、10、15…60），理论上可拥有 1（初始）+ 12 = 最多 13 个技能，或通过多次「加强」深化少量技能。

## 五、技能释放控制

### 5.1 当前阶段（MVP）

| 规则 | 说明 |
|------|------|
| **释放方式** | 由 **AI 自动释放** |
| **AI 逻辑** | 采用简单规则：如按预设优先级选择技能、资源充足时释放、目标选择（仇恨最高/血量最低等） |
| **玩家影响** | 玩家通过「选择学哪些技能、加强哪些技能」间接影响战斗表现 |

### 5.2 后续扩展（策略配置）

| 规则 | 说明 |
|------|------|
| **策略配置** | 玩家可自定义技能使用策略 |
| **可配置项** | 技能释放优先级、目标选择规则（仇恨最高/血量最低/随机）、触发条件（如 HP 低于 X% 时治疗） |
| **与核心循环衔接** | 对应「布局 (Strategize)」：为每位角色配置战斗 AI，使策略自动执行 |

- **设计意图**：先实现简单 AI 保证可玩性，后续开放策略配置以深化「布局」乐趣。

## 六、技能分类与结构（预留）

| 类型 | 说明 | 备注 |
|------|------|------|
| 主动技能 | 消耗资源、需选择目标、有冷却 | 主要设计对象 |
| 被动技能 | 常驻效果，无需释放 | 后续扩展 |
| 普攻 | 无消耗或低消耗的基础攻击 | 每回合默认可用 |

- 具体技能列表、数值公式、冷却与消耗等，见下方「技能池设计」章节。

## 七、与核心循环的衔接

| 循环阶段 | 技能系统贡献 |
|----------|--------------|
| **组建 (Assemble)** | 招募时选择初始技能；等级成长时选择加强或新技能 |
| **布局 (Strategize)** | 当前为 AI 自动释放；后续为玩家配置技能策略 |
| **放置 (Engage)** | 技能在战斗中自动执行，影响伤害、治疗、仇恨等 |
| **分析 (Analyze)** | 战斗日志与数据统计展示技能使用频率、伤害占比等 |
| **优化 (Optimize)** | 根据数据调整技能搭配、加强优先级、策略配置 |

## 八、技能池设计（职业）

### 8.1 战士 (Warrior)

#### 8.1.1 怒气机制 (Rage)

| 规则 | 说明 |
|------|------|
| **初始怒气** | 战斗开始时怒气为 0 |
| **怒气上限** | 固定 100，不随等级变化 |
| **怒气获得** | 受到伤害: +1 怒气/2 点伤害（至少 1 点）；造成伤害: +1 怒气/4 点伤害；部分技能可额外生成怒气 |
| **怒气消耗** | 使用消耗怒气的技能时扣除；怒气不足时无法使用该技能 |
| **脱战** | 战斗结束后怒气归零 |
| **休息** | 休息阶段不恢复怒气 |

- 伤害公式沿用「职业与属性设计」2.2.3；SkillCoeff 为技能倍率。

#### 8.1.2 战士三派系（WoW 经典版对应）

| 派系 | 英文 | 定位 | 典型技能 |
|------|------|------|----------|
| 武器 | Arms | 单体输出、控制、战术 | 英勇打击、撕裂、雷霆一击、顺劈斩、致死打击、斩杀、断筋、横扫攻击、压制、猛击、战斗怒吼、反击风暴、冲锋 |
| 狂暴 | Fury | 多目标、爆发、续航 | 怒击、旋风斩、嗜血、狂怒、猛击、狂暴打击、血性狂怒、狂暴之怒、死亡之愿、怒气澎湃、泰坦之握 |
| 防护 | Protection | 坦克、仇恨、减伤 | 破甲（刷新时额外伤害）、嘲讽、盾牌猛击、复仇、盾牌格挡、震荡猛击、盾墙、挫志怒吼、破釜沉舟、挑战怒吼、盾牌壁垒 |

- **战斗怒吼**：WoW 中为全职业基础技能，本作归入**武器**系（战术指挥、团队增益）。
- **反击风暴**：WoW 经典版为**武器**系天赋，非狂暴。
- 每个技能选择窗口的 3 个选项分别来自上述三派系；等级越高，技能越强。

#### 8.1.3 初始技能（招募时 3 选 1，分属三系，最基础）

初始仅 1 个技能，需在单人场景下也有价值；三者定位区分清晰、相对平衡。

| 派系 | 技能 | 英文 | 消耗 | 冷却 | 效果 | 加强效果 |
|------|------|------|------|------|------|----------|
| 武器 | 英勇打击 | Heroic Strike | 15 怒气 | 0 | 对单体造成物理伤害，倍率 1.2；武器系标准输出 | 倍率 +0.2/次（最高 1.8，最多 3 次） |
| 狂暴 | 嗜血 | Bloodthirst | 20 怒气 | 0 | 对单体造成物理伤害，倍率 1.2，并恢复造成伤害 15% 的生命值；狂暴系续航输出，与武器纯伤害区分 | 倍率 +0.1、吸血 +5%/次（最多 3 次） |
| 防护 | 破甲 | Sunder Armor | 15 怒气 | 0 | 对单体造成物理伤害（倍率 0.8），并使目标护甲 -8，持续 **3 回合**；若目标已有破甲 debuff，则刷新持续时间且本次伤害倍率 +0.3（共 1.1）；高仇恨 | 可叠加层数：每增强 1 次最大层数 +1（最高 4 层），每次施放叠加 1 层并刷新持续时间（最多 3 次） |

- **嗜血**：武器与狂暴初始均为纯伤害时过于相似；改为**嗜血**，突出狂暴系续航（吸血），与武器系纯输出区分。
- **破甲**：持续改为 3 回合；**已有破甲时**可刷新持续时间且本次伤害 +0.3 倍（共 1.1），使反复破甲有收益。

#### 8.1.4 等级解锁技能（5 的倍数时，加强已有 或 3 选 1 学新，分属三系，随等级递进增强）

**等级 5（入门）**

#### 8.1.6 战士初始技能增强（实现规格）

玩家在 5 级、10 级等技能选择窗口选择「强化已有技能」时，`hero.skillEnhancements[skillId].enhanceCount` 递增。**每个技能最多强化 3 次**（enhanceCount 上限 3）。战斗执行技能时，需根据 `enhanceCount` 计算**增强后的技能参数**，再参与伤害/治疗/debuff 计算。以下为战士三个初始技能的增强公式（实现时按 enhanceCount 应用）。

| 技能 ID | 技能名 | 基础参数 | 增强类型 | 公式 | 上限 |
|---------|--------|----------|----------|------|------|
| heroic-strike | 英勇打击 | coefficient: 1.2 | 倍率 | coefficient = 1.2 + enhanceCount × 0.2 | 1.8 |
| bloodthirst | 嗜血 | coefficient: 1.2, healPercent: 0.15 | 倍率 + 吸血 | coefficient = 1.2 + enhanceCount × 0.1；healPercent = 0.15 + enhanceCount × 0.05 | 1.5；0.30 |
| sunder-armor | 破甲 | baseCoefficient: 0.8, refreshCoefficient: 1.1, perStackArmorReduction: 8, debuffDuration: 3 | 可叠加层数 | maxStacks = 1 + enhanceCount；每次施放叠加 1 层并刷新持续时间 | 4 层 |

**说明**：

- **英勇打击**：每次增强 +0.2 倍率，最高 1.8；enhanceCount = 3 时达到上限（1.2 + 0.6 = 1.8）。
- **嗜血**：每次增强同时提升伤害与回血：倍率 +0.1、吸血 +5%；3 次增强后 coefficient = 1.5，healPercent = 0.30（30%）。倍率增幅低于英勇打击，因另有吸血增益。
- **破甲**：增强后支持**叠加层数**。基础 1 层（每层 -8 护甲）；每增强 1 次，最大层数 +1（最高 4 层）。每次施放破甲时：目标已有破甲 debuff 则叠加 1 层（不超过 maxStacks），并**刷新持续时间**至 3 回合；无 debuff 则施加 1 层。总护甲削减 = 8 × 当前层数。

**实现要点**：

- 战斗前或执行技能时，根据 `hero.skillEnhancements[skillId]?.enhanceCount ?? 0` 计算增强后的技能参数；enhanceCount 超过 3 时按 3 处理。
- 破甲 debuff 结构需支持 `stacks` 字段；`armorReduction = perStackArmorReduction × stacks`。
- 技能描述（effectDesc）在 UI 中应反映增强后的数值，便于玩家感知。

**等级解锁技能的增强**：5 级及以上的顺劈斩、旋风斩、嘲讽等，其增强公式在 8.1.4 各等级表格的「加强效果」列中已有描述，实现时按相同模式扩展；**同样遵循每技能最多强化 3 次**。本小节仅规范战士初始三个技能的增强。

# 装备系统设计详细说明

## 一、设计思路概述

装备系统借鉴《暗黑破坏神2》的经典设计，防具参考《流放之路》的护甲/抗性双属性体系。核心围绕**装备部位与类型**、**防具防御属性**、**品质与词缀**、**层级与掉落**、**属性要求与随机性**五个维度展开。

## 二、装备部位与类型 (Equipment Slots & Types)

### 2.1 武器槽位

| 槽位 | 英文标识 | 可装备类型 | 说明 |
|------|----------|------------|------|
| 主手 | MainHand | 单手武器 | 可与副手组合使用 |
| 副手 | OffHand | 盾牌、法术宝珠（Orb） | 持主手单手武器时可装备；双手武器占用主手+副手 |
| 双手 | TwoHand | 双手武器 | 装备于主手槽位，占用主手与副手；副手显示「—」不可装备，UI 不单独显示 TwoHand 槽位 |

- **主手 + 副手**：主手装备单手武器时，副手可装备盾牌（物理/坦克向）或法术宝珠 Orb（法术向，被动持握，仅提供属性加成，不造成武器伤害）。

- **双手武器**：装备双手武器时，主手与副手均被占用，无法再装备盾牌或法系副手。

### 2.3 防具防御属性与属性要求（参考流放之路）

防具（头盔、盔甲、手套、鞋子、腰带）具有**物理护甲**与**法术抗性**两个防御属性，每件防具可侧重其一或两者兼顾：

| 防御属性 | 英文标识 | 效果 | 典型装备侧重 |
|----------|----------|------|--------------|
| 物理护甲 | Armor | 减免所受物理伤害 | 偏物理防护的防具，护甲值高、抗性较低 |
| 法术抗性 | Resistance | 减免所受魔法伤害 | 偏魔法防护的防具，抗性高、护甲较低 |

**属性要求与装备偏向**：

| 装备偏向 | 主要属性要求 | 说明 |
|----------|--------------|------|
| 偏物理防护 | 力量、敏捷 | 高护甲防具通常需要力量或敏捷达到一定数值才能穿戴 |
| 偏魔法防护 | 智力、精神 | 高抗性防具通常需要智力或精神达到一定数值才能穿戴 |
| 均衡型 | 混合要求 | 护甲与抗性兼顾的防具，可能对多种属性有要求 |

- **设计意图**：通过属性要求自然引导构筑方向——物理职业（战士、盗贼等）倾向堆力量/敏捷，易满足高护甲防具；法系职业（法师、牧师等）倾向堆智力/精神，易满足高抗性防具。无硬性职业限制，玩家可通过属性分配灵活选择防具类型。

|------|----------|----------|------|
| 普通 | Normal | 0 | 白装，无词缀，仅基础属性 |
| 魔法 | Magic (Blue) | 1–2 个词缀 | 蓝装，随机 1 或 2 个词缀；**词缀随机范围更大** |
| 稀有 | Rare (Yellow) | 3–4 个词缀 | 黄装，随机 3 或 4 个词缀；词缀数量多但单条随机范围较窄 |
| 暗金 | Unique | 固定属性 + 特殊效果 | 固定词缀组合，具有独特特效 |

- **词缀来源**：魔法/稀有装备从词缀池中随机抽取前缀、后缀；暗金装备拥有预设的固定属性与特殊效果。
- **蓝装 vs 黄装**：蓝装词缀数量少（1–2 条），但**单条词缀的随机范围更大**；黄装词缀多（3–4 条），单条范围较窄。因此**极品蓝装可能优于普通黄装**，黄装并非一定更好，后期玩家仍会对蓝装保持关注与期待。
- **透明化**：玩家获得装备后，可查看该装备各属性的**随机范围**，判断是否为「极品」 roll。

## 四、装备层级与怪物掉落 (Item Tier & Drop)

### 4.1 装备层级（类似暗黑2）

| 层级 | 英文标识 | 说明 | 掉落条件 |
|------|----------|------|----------|
| 普通 | Normal | 基础层级，属性与词缀数值较低 | 任意等级怪物 |
| 拓展 | Exceptional | 中级层级，同词缀加成更高 | 怪物等级超过阈值后开始掉落 |
| 精英 | Elite | 最高层级，同词缀加成最高 | 怪物等级超过更高阈值后开始掉落 |

- **层级与词缀**：同一词缀在不同层级装备上，数值范围不同；精英 > 拓展 > 普通。
- **怪物等级阈值**：具体数值可配置，如怪物等级 ≥ 20 可掉拓展，≥ 40 可掉精英。

### 4.2 词缀层级

词缀本身也分为三个级别，与装备层级对应：

| 词缀级别 | 说明 | 数值范围 |
|----------|------|----------|
| 普通级 | 低等级词缀 | 加成较小 |
| 拓展级 | 中级词缀 | 加成中等 |
| 精英级 | 高级词缀 | 加成最大 |

- **对应关系**：普通装备仅能出现普通级词缀；拓展装备可出现普通级与拓展级；精英装备可出现全部三个级别。
- **同类词缀**：如「+力量」，普通级 +1~3，拓展级 +4~8，精英级 +9~15（数值为示例，可配置）。

### 4.3 底材名称表 (Base Item Names)

每个部位有 **6 个底材族（F1–F6）**，每族包含 **普通 / 拓展 / 精英** 三个层级，共 **18 个底材 / 部位**。

- **底材族 F1–F6**：代表同一类型装备在不同力量层次的形态；F1 最基础，F6 最强大。
- **层级分布**：同族内，**普通**对应 Lv 1–20，**拓展**对应 Lv 21–40，**精英**对应 Lv 41–60；三件装备分别在游戏早、中、晚期掉落，等级要求与属性加成逐层显著提升。
- **命名规律**：同族内 普通 → 拓展 → 精英 听起来越来越强悍；F1 → F6 各族整体由朴素递进至史诗。

### 4.4 底材基础属性表 (Base Item Stats)

每个底材按**层级**分三组列出，各层级有独立的等级要求区间，属性要求和加成随层级显著提升。数值为白板基础范围（未附词缀），词缀额外叠加。

**设计规律：**
- **防具普通级 (Lv 1–20)**：偏物理防护，高护甲低抗性，需**力量**
- **防具拓展级 (Lv 21–40)**：偏魔法防护，高抗性低护甲，需**智力**
- **防具精英级 (Lv 41–60)**：均衡防护，护甲与抗性各半，需**力量+智力**
- **盾牌**三层级均提供护甲+格挡率，全程需力量，强度随层级递增
- **2H 武器**伤害约为同族同层级 1H 的 1.5 倍，属性要求更高
- **匕首与弓**需敏捷，法系武器需智力，其余物理武器需力量
- 同层级内 F1→F6 等级要求与加成按约 4 级递增

## 六、词缀随机范围与可见性 (Affix Roll & Visibility)

### 6.1 随机范围

- 词缀属性在**预设范围内随机**生成，类似暗黑2。
- **品质差异**：蓝装单条词缀的随机范围**更大**（如 +5~18 护甲），黄装单条范围较窄（如 +6~10 护甲）；蓝装 roll 出极品的上限更高，黄装则依赖词缀数量取胜。
- 例如：蓝装「+5~18 护甲」roll 出 17 时，可能优于黄装「+6~10 护甲、+3~5 抗性、+1~2 力量」的平庸 roll。

### 6.2 可见性

| 时机 | 可见内容 |
|------|----------|
| 获得装备后 | 显示该装备**各属性的随机范围**（如 +5~12 护甲），以及当前 roll 出的实际数值 |
| 装备对比 | 对比两件同类装备时，可看到各自的 roll 值与范围，判断是否为「极品」 |

- **设计意图**：增加「打到好装备」的惊喜感与成就感；玩家可明确判断「这件装备是否值得保留」。

### 6.3 装备命名规则 (Item Naming)

#### 6.3.1 命名格式

| 品质 | 命名格式 | 示例 |
|------|----------|------|
| 普通 (白) | **底材名** | Cap, Leather Armor, Short Bow |
| 魔法 (蓝) | **前缀 + 底材 + 后缀**（1 词缀时仅前缀或仅后缀） | Sturdy Cap of the Bear, Mighty Crown |
| 稀有 (黄) | **主前缀 + 底材 + 主后缀 + ", the [称号]"** | Sturdy Crown of the Bear, the Veteran |
| 暗金 (金) | **固定名称**（预设，不随词缀变化） | 见暗金装备表（待扩展） |

#### 6.3.2 蓝装命名细则

- **1 前缀 0 后缀**：`前缀名 + 底材名`，如 `Sturdy Cap`、`Mighty Long Sword`
- **0 前缀 1 后缀**：`底材名 + of + 后缀名`，如 `Cap of the Bear`、`Long Sword of Striking`
- **1 前缀 1 后缀**：`前缀名 + 底材名 + of + 后缀名`，如 `Sturdy Cap of the Bear`

#### 6.3.3 黄装命名细则

黄装有 3–4 条词缀，无法全部放入名称。采用**主词缀 + 称号**方式：

1. **主词缀选取**：从所有前缀中取**词缀层级最高**的一条作为主前缀；从所有后缀中取**词缀层级最高**的一条作为主后缀（层级：精英 > 拓展 > 普通；同层级时按词缀池顺序或随机）。
2. **主名格式**：与蓝装相同，`主前缀 + 底材 + of + 主后缀`。
3. **稀有称号**：在主名后追加 `, the [称号]`。称号从预设池中**根据词缀数量或随机**选取，用于标识「多词缀稀有装」，与蓝装形成视觉区分。
4. **称号池示例**（可配置）：Veteran、Champion、Glory、Bane、Favor、Warden、Sage、Storm、Flame、Frost 等；每件黄装生成时随机或按规则选取一个。

**示例**：
- 蓝装：`Sturdy Crown of the Bear`（1 前 1 后）
- 黄装：`Mighty Crown of the Titan, the Veteran`（2 前 2 后，主词缀为 Mighty + of the Titan，称号 Veteran）

- **详情展示**：装备详情 Modal 中列出全部 3–4 条词缀及 roll 值，名称仅作概览与辨识。

## 七、词缀池设计 (Affix Pool) · 战士验证版

> 当前词缀池以**战士构筑**为设计核心，完成最小化验证。后续扩展至其他职业时，在此基础上追加对应的专用词缀即可。

### 7.1 词缀体系概述

| 维度 | 规则 |
|------|------|
| **类型** | 前缀 (Prefix) / 后缀 (Suffix)；每件装备最多各 1 个前缀、1 个后缀 |
| **品质上限** | 魔法（蓝）最多 2 条词缀（1 前 + 1 后）；稀有（黄）最多 4 条（最多 2 前 + 2 后） |
| **词缀层级** | 普通（掉落自 Lv 1–20 怪物）/ 拓展（Lv 21–40）/ 精英（Lv 41–60） |
| **分类** | **通用词缀**：可出现在所有部位；**专用词缀**：仅出现在特定 1–3 个部位 |

**蓝装 vs 黄装词缀范围规则：**

| 装备品质 | 随机范围规则 | 说明 |
|----------|------------|------|
| 蓝（魔法）| 下限 = max(1, floor(基准×0.7))，上限 = ceil(基准×1.3) | 范围更宽，极品蓝可超越普通黄装；词缀值至少 +1，不出现 +0 |
| 黄（稀有）| 基准范围（表中数值）| 范围稳定，靠多条词缀叠加取胜 |

> 表中所有数值为**黄装基准范围**，蓝装按上方规则换算。

## 八、掉落率与节奏控制 (Drop Rate)

| 设计原则 | 说明 |
|----------|------|
| **低掉率** | 装备整体保持较低掉落率，减少垃圾装备数量 |
| **节约时间** | 玩家无需频繁处理大量无用装备，降低背包管理负担 |
| **惊喜感** | 每次掉落装备时，期待感与成就感更强 |

- **可配置**：具体掉落概率可通过配置调节，便于平衡迭代。
- **与核心循环衔接**：对应「优化 (Optimize)」—— 装备稀缺使每次获得都更有价值，强化构筑与策略的深度。

### 8.1 成色与寻宝值 (Quality & Magic Find)

**核心原则：装备成色与所在区域无关，仅由寻宝值（Magic Find，简称 MF）决定。**

- **寻宝值（MF）**：角色或队伍的一项隐性/显性属性，可通过装备词缀、技能、Buff 等方式提升。
- **成色独立于区域**：即便在等级最低的区域（如 Lv 1 地图），理论上也存在极小概率掉落蓝装、黄装乃至暗金装备；高等级区域不会「锁定」成色，低等级区域也不会「禁止」高品质产出。
- **惊喜感设计**：玩家在练号或低级刷图时偶然获得一件黄装甚至暗金，是系统刻意保留的「彩票时刻」，强化长期留存动力。

| MF 影响维度 | 说明 |
|-------------|------|
| 蓝装（Magic）概率 | MF 直接提升白装升级为蓝装的概率 |
| 黄装（Rare）概率 | MF 提升蓝装升级为黄装的概率（层叠计算） |
| 暗金（Unique）概率 | MF 对暗金的提升效果按折扣系数计算，收益递减（后续拓展） |
| 装备层级 | **不受 MF 影响**；装备层级仍由怪物等级决定（见 4.1） |

> **设计说明**：装备层级（普通/拓展/精英底材）依然与区域挂钩，以保证高等级区域的核心价值；但装备「成几色」完全由 MF 控制，两个维度正交，互不干扰。这样低级玩家在早期区域也有动力刷 MF 词缀、追求高品质装备，而高级区域的吸引力则来自更高层级底材与词缀数值上限。

### 8.2 怪物层级对掉落的影响 (Monster Tier Drop Modifier)

怪物层级（普通 / 精英 / 首领）影响装备掉落概率与成色分布，与 MF 叠加生效：

| 怪物层级 | 装备掉落概率 | 成色分布（相对普通怪） | 特殊规则 |
|----------|--------------|------------------------|----------|
| 普通 (Normal) | 基准概率 | 基准 | 无 |
| 精英 (Elite) | **更高** | **蓝装 / 黄装概率更高** | 无 |
| 首领 (Boss) | **最高** | **蓝装 / 黄装 / 暗金概率更高** | **必然掉落 ≥ 1 件品质至少为魔法（蓝）的装备** |

- **掉落概率**：精英怪的装备掉落概率高于普通怪；Boss 的装备掉落概率最高。具体倍数可配置。
- **成色加成**：在 MF 基础上，精英与 Boss 的成色 roll 有额外加成——同场战斗中出现蓝装、黄装乃至暗金的概率更高。
- **Boss 保底**：击败 Boss 时，**至少掉落 1 件**品质为魔法（蓝）或以上的装备；若 Boss 本身掉落多件，则其中至少一件为蓝装及以上；若 Boss 按概率只应掉 0 件，则强制补发 1 件蓝装。
- **与 MF 叠加**：怪物层级提供的加成与玩家 MF 乘算，两者共同决定最终掉落结果。
- **戒指与项链成色下限**：戒指、项链无底材基础属性，白色成色无任何加成；因此**仅掉落蓝色及以上成色**，若 roll 出白色则强制提升为蓝色。

## 九、与核心循环的衔接

| 循环阶段 | 装备系统贡献 |
|----------|--------------|
| **组建 (Assemble)** | 招募角色时考虑其职业与装备类型限制 |
| **放置 (Engage)** | 战斗胜利后掉落装备，怪物等级影响装备层级与词缀级别 |
| **分析 (Analyze)** | 通过装备词缀与 roll 值评估角色构筑强度 |
| **优化 (Optimize)** | 更换装备、追求极品 roll、根据属性与构筑选择护甲/抗性偏向的防具 |

- 装备词缀系统与「流放式强化系统」（通货道具）可结合：通过通货重塑或增强词缀。
- 具体规则见「制作与强化」章节（待扩展）。

# 背包系统设计详细说明

## 一、设计思路概述

背包系统为简化设计，核心目标是让玩家能够管理战斗掉落的装备与物品，同时通过「出售」机制提供清理背包的途径，降低管理负担。

## 二、背包基础规格

| 属性 | 数值 | 说明 |
|------|------|------|
| 栏位数量 | 100 | 每个玩家账户固定拥有 100 个背包栏位 |
| 栏位类型 | 统一格式 | 每个栏位存放一件装备或物品（不区分类型，不堆叠） |
| 扩展 | 预留付费扩展 | 初期固定 100 格；后续可通过付费解锁更多栏位 |

## 三、物品获取与背包满载规则

- **获取来源**：战斗胜利掉落；商店购买（见「商店系统设计详细说明」）；后续可能包含任务奖励等（待扩展）。
- **背包满载**：当背包已有 100 件物品时，新掉落的装备与物品**无法进入背包**，直接丢弃。
- **满载提示**：背包满时，战斗日志或顶部栏显示提示（如 "Inventory full — loot discarded!"），提醒玩家及时清理。
- **战斗不受影响**：背包满不影响战斗流程、经验与金币的正常获取；仅装备掉落被跳过。

## 四、出售物品

| 规则 | 说明 |
|------|------|
| 操作方式 | 玩家在背包界面点击物品，可选择「出售」 |
| 出售结果 | 物品从背包移除，获得对应金币；金币立即计入玩家账户 |
| 出售价格 | 由物品品质、层级与部位共同决定：品质基础价（普通 8 / 魔法 25 / 稀有 60 / 暗金 150）× 层级系数（普通 1 / 拓展 2 / 精英 4）× 部位系数（主手/双手 1.5、副手 1.3、护甲 1.2、头盔 1.1、护腕/靴子 1、腰带 0.9、项链/戒指 1.1）；可配置 |
| 批量出售 | 暂不支持，每次出售单件（后续可扩展） |
| 无法撤销 | 出售操作不可撤销，物品不可找回 |

## 五、背包 UI 入口

- **入口位置**：顶部栏提供背包按钮/图标（与商店按钮并排），点击后以 Modal 弹窗展示背包内容。
- **背包 Modal 布局**：仅展示有物品的格子（无空位占位）；无物品时显示「No items in backpack」；每格宽度随物品名称长度变化；点击单格弹出物品详情（含出售按钮）。
- **栏位计数**：Modal 顶部显示「N / 100」，玩家可随时查看剩余空间。

## 六、与核心循环的衔接

| 循环阶段 | 背包系统贡献 |
|----------|--------------|
| **放置 (Engage)** | 战斗掉落装备存入背包；背包满时不再接收新掉落，推动玩家回访清理 |
| **优化 (Optimize)** | 玩家查看背包，将好装备穿戴至英雄，将无用装备出售换取金币 |
| **经济 (Economy)** | 出售装备是金币的重要来源之一，形成「战斗掉落 → 出售垃圾 → 金币积累」的资源闭环；商店提供金币消耗途径，形成「战斗/出售 → 金币 → 商店赌博」的闭环 |

# 商店系统设计详细说明

## 一、设计思路概述

商店借鉴《暗黑破坏神2》的**赌博 (Gambling)** 功能：玩家花费金币购买各种「未鉴定」的装备，购买后自动执行鉴定，可能获得该装备的普通、魔法、稀有乃至暗金等各成色版本。核心乐趣在于「花钱开盲盒」的期待感与惊喜感，为金币提供稳定的消耗途径，与战斗掉落、出售装备形成经济闭环。

## 二、购买与鉴定流程

| 阶段 | 说明 |
|------|------|
| **选择部位** | 玩家在商店界面选择要购买的装备部位；武器与副手按类型细分（见「商店可购买槽位」） |
| **支付金币** | 扣除对应部位的价格，金币不足时无法购买 |
| **生成未鉴定装备** | 系统按规则生成一件「未鉴定」装备（部位、底材、等级已确定，品质与词缀未确定） |
| **自动鉴定** | 购买后**立即自动执行鉴定**，复用与怪物掉落相同的品质与词缀随机逻辑 |
| **进入背包** | 鉴定后的装备直接进入玩家背包；若背包已满则按背包满载规则处理（丢弃并提示） |

- **设计说明**：购买即鉴定，无需玩家手动鉴定。鉴定逻辑与怪物掉落完全一致，保证规则透明、实现可复用。

## 三、装备等级与层级

| 规则 | 说明 |
|------|------|
| **等级上限** | 商店出售的装备等级 **不超过玩家小队角色的最高等级**；若小队为空，则默认等级为 1 |
| **层级与底材** | 根据装备等级，按「装备层级与怪物掉落」规则确定可选的层级（普通/拓展/精英）与底材族；与怪物掉落逻辑一致 |
| **等级基准** | 与设计文档中「等级相关机制统一规则」一致：以队伍中最高英雄等级为基准 (baseLevel) |

- **设计意图**：避免玩家通过商店获得远超当前进度的装备，保持与战斗掉落的节奏一致；同时保证商店装备始终「可用」。

## 四、品质与词缀（复用掉落逻辑）

商店装备的品质（普通/魔法/稀有/暗金）与词缀随机，**完全复用**「装备系统设计详细说明」中「八、掉落率与节奏控制」的逻辑：

| 维度 | 说明 |
|------|------|
| **寻宝值 (MF)** | 商店鉴定时，可使用玩家队伍的 MF 参与成色 roll（可配置；若无 MF 则按基准概率） |
| **成色分布** | 与怪物掉落相同的品质概率与升级链（白→蓝→黄→暗金） |
| **词缀** | 魔法/稀有装备从词缀池随机抽取；暗金装备使用固定属性与特效 |
| **戒指/项链** | 戒指、项链仅产出蓝色及以上成色，与掉落规则一致 |

- **可配置**：商店是否受 MF 影响、成色概率是否与掉落完全一致或略有调整，均可通过配置调节。

## 五、价格设计

| 规则 | 说明 |
|------|------|
| **按部位定价** | 不同装备部位有不同基础价格；可与出售价格公式中的「部位系数」挂钩，保持一致性 |
| **等级影响** | 装备等级越高，价格越高；公式：**价格 = 基础价 × (1 + 等级 × 0.08)**，等级系数 0.08 |
| **透明化** | 购买前显示该部位、该等级区间的价格范围或具体价格，避免「盲买」 |

- **平衡目标**：价格应使商店成为「有成本的选择」，既消耗金币又不过于廉价，与战斗掉落形成互补而非替代。
- **示例**：Helm 基础价 250，1 级约 270 金，10 级约 450 金；2H 武器基础价 600，1 级约 648 金。

### 行动中高亮

战斗过程中，正在行动的角色（英雄或怪物）卡片会短暂放大（scale 1.08）并带有**绿色发光边框**，以直观表示该角色正在执行行动。受到攻击且命中的目标会显示**红色边框**和**受伤闪烁效果**（红色背景快速淡出）；若攻击未命中则无效果。

### 面板伤害/治疗数字动画

角色或怪物受到伤害或治疗时，在其卡片面板上显示浮动数字动画，提升感官反馈：

| 场景 | 显示内容 |
|------|----------|
| 受到普通攻击伤害 | 显示 **-X**（红色），X 为实际伤害值 |
| 受到技能伤害 | 显示**技能名称** + **-X**（红色），X 为实际伤害值 |
| 受到技能治疗 | 显示**技能名称** + **+X**（绿色），X 为治疗量 |

动画效果：数字自卡片中心向上浮动并逐渐淡出，约 1.2 秒后消失。伤害数字为红色 (#ff6666)，治疗数字为绿色 (#5cb85c)；技能名称使用技能色并斜体显示。

## 战斗日志增强

### 战斗流程日志

每场战斗在日志中按以下顺序展示，各场战斗之间以分割线和换行分隔：

1. **地图进入描述**（仅进入新地图时）：每次进入新地图（首次加载、切换地图、击败 BOSS 解锁下一张）时，战斗日志先显示该地图的简短描述（如 "Arriving at Elwynn Forest: A peaceful woodland where young adventurers begin their journey..."），以带边框的区块呈现，停顿约 1.8 秒后再开始第一场遭遇
2. **遭遇说明**：战斗开始时显示"Your adventure party encountered XXX!"（Boss 遭遇追加 "fearsome"），以金色斜体呈现；其中怪物名称按 Tier 着色，与战斗日志中怪物姓名颜色一致（Normal/Elite/Boss）
3. **回合日志**：逐条动画显示每次行动，包含详细伤害计算过程
4. **战斗总结**：胜利/失败/平局的总结性描述，含击杀怪物数、回合数、奖励信息
5. **休息阶段**：胜利后显示休息过程，包含 HP/MP 恢复进度，直至"Rest complete"
6. **分割线**：下一场战斗开始前插入视觉分割线

### 姓名颜色

- 战斗日志中，玩家角色姓名统一使用其职业对应的 WoW 职业色（与英雄卡片一致）
- 怪物姓名按 Tier 使用不同颜色：Normal (#66aa88)、Elite (#cc88ff)、Boss (#ff6644)
- 怪物作为攻击者和被攻击者时颜色保持一致

### 伤害数字颜色

| 伤害类型 | 颜色 | 暴击表现 |
|----------|------|----------|
| 物理伤害 | 白色 (#dddddd) | 加粗 + CRIT! 标记 |
| 魔法伤害 | 蓝色 (#44aaff) | 加粗 + CRIT! 标记 |

### 出手顺序与敏捷

- **敏捷显示**：角色或怪物出手时，战斗日志中显示其敏捷数值，格式为 `(AGI N)`，紧跟在出手者姓名后，便于玩家清晰看到「敏捷高者先出手」的原则

### 战斗日志可读性

- **回合标记 [Rx]**：使用较高对比度颜色（#66aa88）与浅色背景，便于识别
- **连接词**（used、on、for）：使用 #88bb99 等可读色，避免与背景混淆
- **伤害类型**（physical）、（magic）：使用 #99ccaa 与浅色背景，与计算详情行区分
- **普通攻击**：战斗日志中普通攻击（basic）显示为白色 (#ffffff)，与技能名称（技能色）区分
- **技能名称与伤害**：技能使用时，技能名称与造成的伤害数值分别使用不同颜色显示，便于快速识别

### 伤害计算详情

每条伤害日志下方附带浅绿色 (#88aa88) 计算详情行，格式为：
`ATK(raw) - Armor(defVal) = final` 或 `ATK(base) x 1.5 - Armor(defVal) = final`（暴击时）
- ATK(raw)：攻击方原始伤害
- Armor(defVal)/Resist(defVal)：防守方护甲或抗性数值
- final：最终伤害

被攻击方 HP 变化：在计算详情下方以 HP 色显示 `{被攻击者姓名} HP: before -> after/max`，便于复盘时追踪目标血量变化。

### 暴击系统

- 英雄暴击率由职业系数决定：PhysCrit = 5 + Agi * k_PhysCrit，SpellCrit = 5 + Int * k_SpellCrit
- 怪物暴击率按 Tier：Normal 5%，Elite 10%，Boss 15%
- 暴击伤害倍率：1.5x

## 战斗自动化

- 无任何手动战斗按钮（已移除 Explore Normal/Elite、Start Encounter、Fight Zone Boss、Recover One Turn）
- **战斗暂停**：战斗日志区域提供暂停/继续按钮，暂停时日志停止滚动、战斗动画暂停，便于玩家详细查看战斗过程
- 组件加载后自动启动战斗循环：遭遇说明 → 战斗 → 日志动画 → 战斗总结 → 自动休息（含日志） → 分割线 → 下一场战斗
- 战胜积累探索度，探索度满时自动挑战 BOSS
- 休息阶段每步间隔 2000ms，每步在日志中显示恢复进度
- 战败后同样进入恢复阶段（"Recovering from defeat..."），逐步恢复所有英雄 HP/MP 至满值后再开始下一场战斗

- **数据可视化设计**: 战斗日志语义着色、HP/资源进度条、探索度进度条

## 显示规格与适配策略

- **目标平台**: 桌面端显示器，不再进行移动端适配。
- **目标比例**: 固定按 16:9 视觉舞台进行布局，界面以该比例为基准设计与实现。
- **最低分辨率**: 1920 x 1080（1080P），低于该分辨率视为不支持运行环境。
- **布局原则**:
  - 所有核心页面以大屏信息密度为优先，不再为窄屏做断点重排。
  - 主界面采用固定舞台（16:9）+ 内容分栏布局，减少滚动依赖。
  - 在超宽或非 16:9 屏幕上，通过留边方式保持核心内容比例一致。
- **交互反馈**: 当检测到分辨率低于 1080P 时，前端展示不支持提示，并引导用户调整分辨率或更换设备。

# 技术规格 (Technical Specification)

## 技术栈总览

| 层级 | 选型 | 说明 |
|------|------|------|
| 后端 | Go | 高并发、单二进制部署、生态成熟 |
| 前端 | Vue 3 | 组件化、响应式、适合复杂数据展示 |
| 数据库 | SQLite | 初期轻量部署，使用 GORM 便于后续迁移至 PostgreSQL |
| 通信 | REST + WebSocket | REST 常规接口，WebSocket 实时推送 |

## 后端 (Backend)

- **语言与框架**: Go + Gin / Fiber
- **选型理由**:
  - 原生并发支持（goroutine），适合多玩家、战斗结算、离线收益计算等场景
  - 单二进制部署，资源占用低，适合中小规模部署
  - Web 框架成熟，便于快速开发 REST API 与 WebSocket 服务
- **关键技术**:
  - WebSocket: gorilla/websocket 或 nhooyr.io/websocket（战斗日志、聊天等实时推送）
  - 离线收益: 服务端按离线时长计算，保证防作弊与数据一致性

## 前端 (Frontend)

- **框架**: Vue 3
- **选型理由**:
  - Composition API 便于组织战斗日志、数据统计等复杂逻辑
  - 响应式系统适合实时战斗面板、挂机收益统计等动态更新
  - 组件化适合拆解主界面、战斗、角色、背包等模块
- **技术建议**:
  - 状态管理: Pinia
  - 路由: Vue Router
  - UI 风格: 复古像素风（见下方界面美术风格）
  - 显示规格: 仅支持桌面端 16:9，最低 1080P，不提供移动端响应式断点

## 数据库 (Database)

- **选型**: SQLite（MVP / Alpha 阶段）
- **适用场景**: 单服部署、玩家量级较小、读写并发不高
- **注意事项**:
  - SQLite 为单写锁，高并发写入时会排队
  - 若 Beta / 正式版预期玩家增多或需多服部署，可迁移至 PostgreSQL
- **ORM**: GORM（支持 SQLite 与 PostgreSQL，迁移时仅需调整连接配置）

## 客户端/服务端架构

- **架构模式**: 前后端分离，前端 SPA + 后端 API 服务
- **离线收益**: 服务端权威计算，客户端仅展示与上报在线/离线状态
- **实时通信**: 战斗日志、聊天等通过 WebSocket 推送；排行榜、交易等可先采用 REST + 定时刷新

## 界面美术风格

- **整体风格**: 黑底绿字怀旧电脑风格（CRT 终端风格），与「复古极简美学」设计理念一致
- **实现要点**:
  - **配色方案**: 
    - 背景：纯黑色 (#000000)
    - 主文字：绿色 (#00ff88)，营造 CRT 显示器效果
    - 边框：绿色，使用 1-2px 实线
    - 强调色：亮绿色 (#00ffaa) 用于高亮和交互元素
    - 错误/警告：红色 (#ff6666) 用于错误，橙色 (#ffcc66) 用于警告
    - **语义色扩展**（用于战斗信息差异化显示）：
      - HP 条与数字：颜色随受伤程度变化，健康(>75%)绿(#44ff88)、受伤(26-75%)黄(#ffdd66)、重伤(1-25%)红(#ff4444)、死亡灰(#888888)
      - MP 条：蓝色 (#44aaff)
      - 物理伤害：橙色 (#ffaa44)
      - 魔法伤害：紫色 (#cc88ff)
      - 技能标注：黄色 (#ffee66)
      - 胜利结果：青色 (#00ffcc)
      - 失败结果：红色 (#ff4444)
      - 金币：金黄色 (#ffcc44)
      - 经验：浅绿色 (#88ffaa)
      - Boss：橙红色 (#ff6644)
  - **滚动条**: 自定义样式匹配暗绿终端主题（6px 窄滚动条，暗色轨道 #0a0a0a，绿色滑块 #1a3a1a，悬浮 #2a5a2a）
  - **字体**: 像素字体（如 Press Start 2P、VT323、Silkscreen）或等宽字体，强化终端感
  - **视觉效果**: 
    - 可添加轻微的文字发光效果（text-shadow）模拟 CRT 荧光
    - 边框使用简洁的实线，避免过度装饰
    - 保持简洁的布局，强化「命令行终端」的怀旧感
  - **图像**: `image-rendering: pixelated` 或 `crisp-edges` 保持像素图不模糊
  - **布局**: 网格化、固定宽高，强化「格子感」和「终端窗口」感

## 防作弊策略

- **核心原则**: 所有关键数值在服务端计算，客户端仅负责展示与输入
- **具体措施**:
  - 战斗结算、装备掉落、离线收益等逻辑全部在服务端执行
  - 客户端不参与任何权威数值计算，仅接收服务端结果并展示
