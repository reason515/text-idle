# 游戏名称
Text Idle

# 高阶理念
一款深度策略型文字放置RPG。游戏致力于将《魔兽世界》的职业战术、《暗黑破坏神》的装备驱动、《流放之路》的物品制作等经典设计精髓，提炼并浓缩于纯粹的文字体验之中。我们摒弃繁杂的图形，旨在打造一个让玩家能心无旁骛、深度钻研策略与构筑（Build）的硬核游戏环境。

# 设计哲学
- **策略优先**: 游戏的核心乐趣源于深度的策略思考，而非频繁的操作。通过离线挂机机制解放玩家的时间，让每一次决策都显得至关重要。
- **透明化设计**: 采用"小数值"系统，所有属性、伤害计算、怪物行为逻辑等底层公式完全对玩家开放。我们鼓励玩家通过数据分析、战斗日志复盘等方式，成为精通游戏机制的大师。
- **致敬经典**: 汲取ARPG与MMORPG黄金时代的灵感，重现"战法牧"铁三角、仇恨机制、惊喜掉落以及装备词缀系统带来的无穷乐趣。

# 核心游戏循环
- **组建 (Assemble)**: 招募并组建一支五人小队。根据职业特性（如坦克、治疗、伤害输出），打造一支独一无二的冒险队伍。
-  **布局 (Strategize)**: 为每位角色精细配置战斗AI。设定技能释放的优先级、触发条件以及团队战术，让你的策略自动执行。
- **放置 (Engage)**: 队伍将依据你的策略自动进行探索与战斗，获取经验、金币与装备。即使玩家离线，探索也从不间断。
- **分析 (Analyze)**: 通过实时的战斗数据面板与详尽的挂机收益统计，精准评估当前策略的效率与短板。
- **优化 (Optimize)**: 使用获取的资源与装备强化你的队伍。调整技能搭配、更新装备、优化战术，以挑战更强大的敌人，追求更高的挂机收益。

# 关键特征
- **深度策略小队**: 经典的"坦克、治疗、DPS"角色定位与仇恨机制，要求玩家深刻理解职业 synergy，构筑平衡且高效的队伍。

## 游戏概述
- **核心愿景 (Core Vision)**
	- 致力于为玩家提供一个纯粹、无负担、可长期沉浸的策略型放置游戏体验。我们旨在让玩家能够轻松利用碎片时间进行深度策略研究与享受游戏乐趣，而非被繁琐操作所束缚。
- **目标用户 (Target Audience)**
    - 追求深度策略与机制研究，乐于进行理论构筑（Theorycrafting）的核心玩家。
    - 热衷于《魔兽世界》、《暗黑破坏神》等经典RPG，享受团队配合与装备驱动乐趣的玩家群体。
    - 忙碌的上班族或学生党，渴望利用碎片时间放松，避免高频操作负担，但仍能体验到游戏深度。
- **独特卖点 (Unique Selling Points)**
    - **极致透明的机制**: 游戏所有底层机制、属性、计算公式完全公开，鼓励玩家深度挖掘，成为真正的策略大师。
    - **自由定制的战术**: 提供高度灵活的作战策略配置，玩家可自由定义队伍AI，实现千变万化的战术组合。
    - **数据驱动的优化**: 详尽的战斗数据统计与分析功能，帮助玩家直观评估策略效果，形成"策略制定-数据验证-迭代优化"的正向循环。

# 核心玩法与循环
- **组建 (Assemble)**: 角色获取与队伍构成
	- 队伍规模随进度逐步解锁：从 1 名角色起步，每推进一张地图增加一个席位，最终最多 5 人，让玩家在职业与机制上循序渐进，避免初期信息过载。
	- 每名新成员加入时，由玩家为其选择职业与初始技能，从而塑造独一无二的队伍构成。
- **布局 (Strategize)**: 战斗策略与 AI 配置
	- 在"坦克、治疗、DPS"定位与仇恨机制的基础上，为每位角色配置战斗 AI：自定义技能（含普攻）的释放条件、目标选择与优先级顺序，使策略自动执行。
	- 策略是否合理需结合战斗日志与数据统计来验证与迭代，形成"配置—观察—调整"的闭环。
- **放置 (Engage)**: 关卡、怪物与离线收益
	- 队伍按当前策略自动探索与战斗，持续获取经验、金币与装备；支持离线挂机，探索不中断。
	- 离线期间按真实战斗逻辑结算，数据统计与装备掉落与在线时一致，保证收益可预期。
	- 离线挂机设有最大累积时长，超时未上线则不再累积收益，鼓励定期回访。
	- 单场战斗的怪物数量多数与队伍人数一致，少数场次会多于或少于，用于调节难度与节奏。
	- 关卡与怪物命名借鉴《魔兽世界》经典地图与生物，强化熟悉感与代入感。
	- 怪物分为普通、精英、首领三档，强度与掉落品质逐级提升。
	- 通过击杀怪物积累关卡进度，达到要求后解锁后续关卡，驱动推进节奏。
	- 关卡探索中可能"偶遇"其他也在当前关卡挂机的玩家，触发 PVP；战斗结果单独统计，战胜可获得荣誉积分，用于兑换稀有道具，并参与荣誉排行榜。
- **分析 (Analyze)**: 战斗日志与数据统计
	- 通过实时战斗面板与挂机收益统计，评估策略效率、定位短板，为优化提供依据。
- **优化 (Optimize)**: 角色成长与装备驱动
	- 用获得的资源与装备强化队伍：升级技能、更换词缀装备、调整战术，以挑战更强敌人、提升挂机收益。

		- **强化规则**

	- **核心资源**: 种类、获取途径

# 关卡、地图与怪物设计详细说明

## 一、地图系统 (Map System)

### 1.1 解锁机制（大秘境式探索）

借鉴《暗黑破坏神》大秘境的设计思路，地图采用**探索度驱动**的推进模式：

| 阶段 | 机制说明 |
|------|----------|
| **初始状态** | 玩家仅能选择**初始地图**进行探索，无法进入后续地图 |
| **刷怪积累** | 在地图上持续战斗、击杀怪物，逐步积累**探索度 (Exploration Progress)** |
| **BOSS 出现** | 探索度达到 100% 时，该地图的**守关 BOSS** 出现 |
| **解锁下一张** | 击败 BOSS 后，解锁下一张地图，并可选择新地图继续推进 |

- **探索度公式**：每击杀一只怪物，根据怪物等级与类型贡献一定探索度；普通怪贡献较少，精英怪贡献较多，BOSS 不计入探索度（为守关目标）。
- **节奏控制**：探索度增长速度可调，用于控制单张地图的游玩时长与推进节奏。

### 1.2 英雄席位与地图解锁绑定

队伍规模与地图进度严格绑定，形成**循序渐进**的成长曲线：

| 地图序号 | 队伍席位 | 说明 |
|----------|----------|------|
| 初始 | 1 人 | 玩家仅能选择 1 位英雄加入队伍 |
| 解锁第 2 张地图 | 2 人 | 每解锁一张新地图，可再招募 1 位英雄 |
| 解锁第 3 张地图 | 3 人 | 同上 |
| 解锁第 4 张地图 | 4 人 | 同上 |
| 解锁第 5 张地图 | 5 人 | 队伍满编，最多 5 人 |

- **设计意图**：避免初期信息过载，让玩家在熟悉单英雄操作后再逐步扩展队伍，同时强化「解锁新地图」的奖励感。

### 1.3 经典地图列表（魔兽世界风格）

采用《魔兽世界》中玩家耳熟能详的经典地图及对应怪物，强化熟悉感与代入感。地图按难度递进排列：

| 序号 | 地图英文名 | 地图中文名 | 典型怪物（示例） | 守关 BOSS |
|------|------------|------------|------------------|-----------|
| 1 | Elwynn Forest | 艾尔文森林 | Kobold, Wolf, Defias | Hogger |
| 2 | Westfall | 西部荒野 | Defias, Harvest Golem, Vulture | VanCleef |
| 3 | Duskwood | 暮色森林 | Skeleton, Worgen, Zombie | Stitches |
| 4 | Redridge Mountains | 赤脊山 | Blackrock Orc, Spider | Kazon |
| 5 | Stranglethorn Vale | 荆棘谷 | Troll, Tiger, Panther | King Bangalash |

- **命名规范**：地图与怪物均采用 WoW 经典译名，便于玩家识别。
- **扩展性**：后续版本可追加更多地图（如 Burning Steppes、Un'Goro 等），保持相同解锁逻辑。

## 二、怪物系统 (Monster System)

### 2.1 怪物等级划分

怪物分为三个等级，强度与机制复杂度逐级提升：

| 等级 | 英文标识 | 强度特点 | 技能使用 | 掉落品质 |
|------|----------|----------|----------|----------|
| 普通 | Normal | 基础属性，无特殊机制 | 仅普攻 | 普通/魔法 |
| 精英 | Elite | 生命/伤害显著提升 | **会使用技能**，需谨慎应对 | 魔法/稀有 |
| 首领 | Boss | 最高强度，多阶段或特殊机制 | 多技能组合，高威胁 | 稀有/独特 |

- **精英及以上**：从精英开始，怪物会使用技能（如打断、AOE、控制等），玩家需根据仇恨、治疗与打断节奏调整策略。
- **Boss**：守关 BOSS 拥有更强技能与机制，是检验队伍构筑与策略的关键节点。

### 2.2 伤害类型

怪物可造成**物理伤害**或**魔法伤害**，与玩家的护甲/抗性体系对应：

| 伤害类型 | 英文标识 | 减伤属性 | 典型怪物示例 |
|----------|----------|----------|--------------|
| 物理 | Physical | 护甲 (Armor) | Wolf, Defias, Orc, Tiger |
| 魔法 | Magic | 抗性 (Resistance) | Mage-type Kobold, Shadow Wolf, Caster Troll |

- **混合型怪物**：部分精英或 BOSS 可同时拥有物理与魔法攻击，考验玩家的双抗配置。
- **透明化**：怪物面板与战斗日志中明确标注伤害类型，便于玩家针对性堆叠护甲或抗性。

### 2.3 怪物属性设计 (Monster Attributes)

怪物采用**简化属性体系**，与英雄共用同一套伤害/减伤公式，保证战斗计算透明且易于配置。

#### 2.3.1 核心战斗属性

| 属性 | 英文标识 | 用途 | 说明 |
|------|----------|------|------|
| 生命值 | HP | 承受伤害，归零即死亡 | 公式见下方，受类型系数与等级系数影响 |
| 物理攻击力 | PhysAtk | 造成物理伤害 | 普攻与物理技能的基础数值 |
| 法术强度 | SpellPower | 造成魔法伤害 | 法术技能的基础数值 |
| 敏捷 | Agility | 出手顺序 | 与英雄共用「敏捷高者先手」规则；同敏捷随机 |
| 护甲 | Armor | 减免所受物理伤害 | 英雄攻击怪物时，怪物护甲参与减伤计算 |
| 法术抗性 | Resistance | 减免所受魔法伤害 | 英雄攻击怪物时，怪物抗性参与减伤计算 |

- **防御公式**：与英雄一致，1 护甲 = 抵消 1 点物理伤害，1 抗性 = 抵消 1 点魔法伤害，无上限。
- **伤害公式**：怪物对英雄造成伤害时，使用英雄的护甲/抗性；英雄对怪物造成伤害时，使用怪物的护甲/抗性。

#### 2.3.2 可选次级属性（精英/BOSS）

| 属性 | 英文标识 | 用途 | 说明 |
|------|----------|------|------|
| 物理暴击率 | PhysCrit | 物理攻击暴击概率 | 普通怪可设为 0 或 5% 基础值 |
| 法术暴击率 | SpellCrit | 法术攻击暴击概率 | 同上 |
| 闪避率 | Dodge | 躲避物理攻击的概率 | 高敏捷型怪物可适当提高 |
| 命中率 | Hit | 攻击命中概率 | 多数怪物可固定为 95% 基础，BOSS 可略高 |

- **简化原则**：普通怪可仅配置核心属性，次级属性使用默认值；精英与 BOSS 按需配置，体现差异化。

#### 2.3.3 伤害类型与属性对应

| 怪物类型 | PhysAtk | SpellPower | PhysRatio | MagicRatio | 示例 |
|----------|---------|------------|-----------|------------|------|
| 纯物理 | 有值 | 0 | 1.0 | 0 | Wolf, Defias, Orc |
| 纯魔法 | 0 | 有值 | 0 | 1.0 | Kobold Geomancer, Caster Troll |
| 混合型 | 有值 | 有值 | 0.5 | 0.5 | 部分精英、BOSS |

- 普攻与技能根据 PhysRatio/MagicRatio 选择使用 PhysAtk 或 SpellPower 参与伤害计算。

#### 2.3.4 属性公式与配置参数

```
HP       = Base_HP * TierMult * (1 + Level * LevelScale_HP)
PhysAtk  = Base_PhysAtk * TierMult * (1 + Level * LevelScale_Atk)
SpellPower = Base_SpellPower * TierMult * (1 + Level * LevelScale_Atk)
Agility  = Base_Agility + Level * LevelScale_Agi
Armor    = Base_Armor * TierMult
Resistance = Base_Resistance * TierMult
```

| 参数 | 说明 | 示例 |
|------|------|------|
| Base_* | 每类怪物的基础值 | 如 Young Wolf: Base_HP=30, Base_PhysAtk=8 |
| TierMult | 类型系数：Normal=1, Elite≈2, Boss≈5 | 可配置 |
| Level | 怪物等级（通常与地图或玩家等级挂钩） | 1–60 |
| LevelScale_* | 等级缩放系数 | 如 LevelScale_HP=0.02 |

- **小数值原则**：1 级普通怪 HP 约 20–50，PhysAtk/SpellPower 约 5–15，与 1 级英雄量级相当；随等级与类型系数放大。

#### 2.3.5 怪物属性示例（1 级普通怪，TierMult=1）

| 怪物 | HP | PhysAtk | SpellPower | Agility | Armor | Resistance | 伤害类型 |
|------|-----|---------|------------|---------|-------|------------|----------|
| Young Wolf | 25 | 8 | 0 | 6 | 2 | 0 | 纯物理 |
| Kobold Miner | 22 | 6 | 0 | 4 | 1 | 0 | 纯物理 |
| Kobold Geomancer | 20 | 0 | 10 | 5 | 0 | 3 | 纯魔法 |
| Defias Trapper | 28 | 7 | 0 | 7 | 2 | 0 | 纯物理 |

- 具体数值以实际平衡为准，此处为量级参考。

### 2.4 怪物强度调节（可配置化）

为便于后续不断调试游戏平衡，怪物强度采用**可配置参数**设计：

| 参数类别 | 参数示例 | 说明 |
|----------|----------|------|
| **基础属性** | HP, PhysAtk, SpellPower, Agility, Armor, Resistance | 每类怪物的基础数值，可单独调节 |
| **等级系数** | LevelScale_HP, LevelScale_Atk, LevelScale_Agi | 随地图/玩家等级变化的缩放系数 |
| **类型系数** | Elite_HP_Mult, Boss_HP_Mult, Elite_Armor_Mult 等 | 精英/BOSS 相对普通怪的倍率 |
| **伤害类型权重** | PhysRatio, MagicRatio | 物理/魔法伤害占比，用于混合型怪物 |
| **次级属性** | PhysCrit, SpellCrit, Dodge, Hit | 精英/BOSS 可选配置 |

- **配置存储**：建议使用 JSON/YAML 或数据表存储，便于策划与程序协同调整。
- **热更新**：若技术可行，支持不重启服务即可更新怪物配置，加速平衡迭代。

## 三、与核心循环的衔接

- **放置 (Engage)**：队伍在选定地图上自动刷怪，探索度持续累积；探索度满后 BOSS 出现，击败即解锁下一张地图。
- **组建 (Assemble)**：每解锁一张新地图，可招募一名新英雄，直至队伍满 5 人。
- **优化 (Optimize)**：根据怪物伤害类型（物理/魔法）调整装备与抗性；根据精英/BOSS 技能调整战术与打断优先级。

# 战斗系统设计详细说明

## 一、回合与出手顺序 (Turn Order)

### 1.1 出手顺序规则

战斗采用**回合制**，每回合内所有参战单位（玩家英雄与怪物）按**敏捷 (Agility)** 从高到低依次行动：

| 规则 | 说明 |
|------|------|
| **敏捷决定顺序** | 敏捷越高的角色（含怪物）越先出手 |
| **同敏捷处理** | 敏捷相同时，出手顺序随机决定 |
| **透明化** | 战斗日志中记录出手顺序与行动结果，便于战后复盘与战术调整（本游戏为放置游戏，战斗过程玩家无法实时操控，仅能通过预先配置的战术策略间接影响） |

- **设计意图**：敏捷不仅影响暴击、闪避，还直接影响行动先后，使高敏捷职业（盗贼、猎人、德鲁伊）在节奏上有优势。

### 1.2 每回合行动次数

| 规则 | 说明 |
|------|------|
| **每回合每角色行动一次** | 每个参战单位（英雄或怪物）每回合仅能执行一次行动（普攻或技能） |
| **行动类型** | 行动可为普攻、主动技能或待机等，由战斗 AI 或玩家策略决定 |

- **小数值与节奏**：单回合单行动保证战斗节奏清晰，便于仇恨、治疗与打断的时序规划。

---

## 二、怪物数量与战斗规模 (Encounter Size)

### 2.1 怪物数量分布

单场战斗中怪物数量与玩家小队规模的关系如下：

| 情况 | 概率 | 说明 |
|------|------|------|
| **与队伍人数相当** | 较大概率 | 如 3 人队伍时，多数场次为 3 只怪 |
| **少于队伍人数** | 较小概率 | 如 3 人队伍时，少数场次为 1–2 只怪，节奏更快 |
| **多于队伍人数** | 较小概率 | 如 3 人队伍时，少数场次为 4–5 只怪，难度提升 |

- **可配置**：具体概率分布（如 70% 相等、15% 少、15% 多）可通过配置调节，便于平衡调试。
- **设计意图**：多数场次势均力敌，少数场次制造节奏变化与难度起伏。

## 三、胜负判定 (Victory & Defeat)

| 结果 | 触发条件 | 后续 |
|------|----------|------|
| **胜利** | 怪物全部死亡 | 进入战后结算（经验、金币、掉落）；进入休息恢复阶段（自动执行） |
| **失败** | 玩家英雄全部死亡 | 战斗结束；不获得本场掉落；**扣除当前地图探索度 10 点（可配置），最低不低于 0**；短暂等待后自动开始下一场战斗 |

- **战败惩罚**：每次战败扣除探索度（默认 10 点，可配置），防止零成本无限刷怪；探索度不会因战败归零，保留已积累的进度。
- **透明化**：胜负判定逻辑对玩家完全公开，便于评估风险与策略取舍。

---

## 四、战后休息与恢复 (Post-Combat Rest)

### 4.1 恢复机制

每场战斗胜利后，队伍自动进入**休息阶段**，无需玩家手动操作，系统逐步恢复所有角色资源，直至完全恢复后自动开始下一场战斗：

| 规则 | 说明 |
|------|------|
| **自动执行** | 休息恢复全程自动进行，玩家无需点击任何按钮 |
| **恢复门槛** | 所有英雄 HP 与 MP（或怒气/能量/集中值等）均恢复至满值后，自动开始下一场战斗 |
| **恢复速度** | HP、MP 的恢复速度受**精神 (Spirit)** 属性以及**装备**等因素影响 |
| **死亡惩罚** | 若有角色在本场战斗中死亡，恢复时间将延长（如恢复速度降低或需额外恢复回合），以体现战斗代价 |
| **时间单位** | 恢复按**回合数**或**时间步**结算（遵循游戏「回合制」设计，不使用秒/分钟等实时单位） |

### 4.2 恢复公式（草案）

| 资源 | 恢复公式（每回合/每步） | 主要影响因素 |
|------|--------------------------|--------------|
| HP | `HP_Regen = Base + Spi * k_HP_Regen + 装备恢复加成` | 精神、装备词缀 |
| MP | `MP_Regen = Base + Spi * k_MP_Regen + 装备恢复加成` | 精神、装备词缀 |

- **怒气/能量/集中值**：脱战后可快速归零或按固定规则重置，具体在资源系统设计时确定。
- **死亡惩罚**：若有 N 名角色在本场战斗中死亡，恢复速度可乘以惩罚系数（如 `1 + N * k_DeathPenalty`），或增加固定额外恢复回合，具体数值可配置调节。
- **透明化**：恢复速度与公式对玩家可见，便于评估「精神」与恢复词缀的收益。

### 4.3 与核心循环的衔接

- **放置 (Engage)**：战斗 → 休息恢复 → 下一场战斗，形成连续的探索节奏。
- **优化 (Optimize)**：堆叠精神或恢复词缀可缩短休息时间，提高挂机效率。

## 五、与仇恨、伤害公式的衔接

- **仇恨机制**：怪物每回合按仇恨值选择目标，详见「仇恨机制」章节。
- **伤害与减伤**：物理/魔法伤害、护甲/抗性减伤公式见「职业与属性设计」2.2.3。
- **状态效果**：Buff/Debuff 对出手顺序、恢复等的影响，见「状态效果」设计。

## 一、职业设计 (Classes)

### 1.4 主属性与成长

- 每个职业有 **主属性 (Primary Attribute)**，升级与装备提供的该属性收益更高。

- **升级自由分配**：每次升级获得 **5 点主属性**，由玩家自由分配至力量、敏捷、智力、耐力、精神中的任意一项。满级（60 级）时累计可获得 `5 × (60 - 1) = 295` 点自由属性。

### 1.5 职业与技能关系

- 职业决定可学习的 **技能池**；玩家从池中选择技能装备到角色。
- 部分技能为职业专属，部分为通用（如普攻变体）。
- 技能获取方式：初始技能、升级解锁、装备/道具解锁。

## 二、核心属性 (Attributes)

### 2.1 属性定义

采用小数值、透明化设计，所有属性对战斗数值的影响公式对玩家完全公开。

| 属性 | 英文标识 | 主要影响 | 典型受益职业 |
|------|----------|----------|--------------|
| 力量 | Strength | 物理攻击力、护甲、格挡值 | 战士、圣骑士 |
| 敏捷 | Agility | 暴击率、闪避、物理命中 | 盗贼、猎人、德鲁伊 |
| 智力 | Intellect | 法术强度、法力上限、法术暴击 | 法师、牧师、术士、圣骑士 |
| 耐力 | Stamina | 生命值上限 | 全职业（坦克尤其重要） |
| 精神 | Spirit | 法力恢复、生命恢复（脱战） | 治疗职业 |

### 2.2 次级属性与资源系统

#### 2.2.1 资源类型 (Resources)

| 资源 | 英文标识 | 适用职业 | 上限公式 | 恢复方式 |
|------|----------|----------|----------|----------|
| 生命值 | HP | 全职业 | 见 2.2.2 | 治疗技能、脱战精神恢复 |
| 法力值 | MP | 法师、牧师、术士、萨满、圣骑士、德鲁伊 | 见 2.2.2 | 战斗内精神恢复、脱战快速恢复 |
| 怒气 | Rage | 战士 | 固定 100，不随等级变化 | 受到伤害获得、造成伤害获得 |
| 能量 | Energy | 盗贼 | 固定 100，不随等级变化 | 固定速率恢复（如每回合 10 点） |
| 集中值 | Focus | 猎人 | 固定 100 | 固定速率恢复、技能消耗 |

- **小数值原则**：怒气/能量/集中值上限固定为 100，便于技能消耗设计（如 20 怒气、30 能量）。
- **资源差异化**：法力职业依赖精神恢复；怒气职业依赖战斗节奏；能量/集中职业依赖时间与节奏。

#### 2.2.2 次级属性公式（小数值版 + 职业差异化）

所有系数采用小数值，使 1 级角色衍生属性在 10–50 量级，满级（60 级）在 100–350 量级，便于心算与透明化。

**通用公式结构**：`次级属性 = Base + 主属性 * k_职业`，其中 `k_职业` 为职业系数，体现职业对某属性的转化效率差异。

##### 基础公式（职业系数见下表）

| 次级属性 | 英文标识 | 通用公式 | 主属性来源 |
|----------|----------|----------|------------|
| 生命值上限 | HP | `HP = 10 + Stam * k_HP + Level * 2` | 耐力 |
| 法力值上限 | MP | `MP = 5 + Int * k_MP + Level * 1` | 智力（仅法力职业） |
| 物理攻击力 | PhysAtk | `PhysAtk = 5 + 主物理属性 * k_PhysAtk` | 力量或敏捷（见职业表） |
| 法术强度 | SpellPower | `SpellPower = 5 + Int * k_SpellPower` | 智力（仅法术职业） |
| 护甲 | Armor | `Armor = Str * k_Armor + 装备护甲` | 力量 |
| 法术抗性 | Resistance | `Resistance = Int * k_Resistance + 装备抗性` | 智力 |
| 物理暴击率(%) | PhysCrit | `PhysCrit = 5 + Agi * k_PhysCrit` | 敏捷 |
| 法术暴击率(%) | SpellCrit | `SpellCrit = 5 + Int * k_SpellCrit` | 智力 |
| 闪避率(%) | Dodge | `Dodge = 5 + Agi * k_Dodge` | 敏捷 |
| 命中率(%) | Hit | `Hit = 95 + Agi * 0.2` | 敏捷（全职业统一） |
| 格挡率(%) | Block | `Block = 装备/技能提供` | 仅持盾职业 |

##### 职业系数表（k 值）

系数设计考虑满级（60 级）时 295 点自由属性全加主属性的极端情况，使攻击力/法术强度控制在 100–250 量级，符合小数值原则。

| 职业 | k_HP | k_MP | PhysAtk 主属性 | k_PhysAtk | k_SpellPower | k_Armor | k_Resistance | k_PhysCrit | k_SpellCrit | k_Dodge |
|------|------|------|----------------|-----------|--------------|---------|--------------|------------|-------------|---------|
| 战士 | 4.0 | - | Str | 0.65 | - | 0.8 | 0.3 | 0.3 | - | 0.2 |
| 圣骑士 | 3.5 | 2.2 | Str | 0.45 | 0.45 | 0.6 | 0.6 | 0.3 | 0.4 | 0.2 |
| 牧师 | 2.5 | 2.8 | - | - | 0.65 | - | 0.8 | 0.3 | 0.6 | 0.2 |
| 德鲁伊 | 3.2 | 2.2 | Agi | 0.5 | 0.45 | 0.4 | 0.6 | 0.6 | 0.5 | 0.4 |
| 法师 | 2.0 | 2.8 | - | - | 0.65 | - | 0.8 | 0.3 | 0.6 | 0.2 |
| 盗贼 | 2.8 | - | Agi | 0.55 | - | 0.2 | 0.3 | 0.7 | - | 0.5 |
| 猎人 | 3.0 | - | Agi | 0.5 | - | 0.3 | 0.3 | 0.6 | - | 0.4 |
| 术士 | 2.8 | 2.8 | - | - | 0.65 | - | 0.8 | 0.3 | 0.6 | 0.2 |
| 萨满 | 3.0 | 2.2 | Agi | 0.4 | 0.45 | 0.3 | 0.6 | 0.5 | 0.5 | 0.3 |

- **设计说明**：坦克职业（战士）HP/护甲系数高；物理 DPS（盗贼、猎人）暴击/闪避系数高；法术职业（法师、牧师、术士）法术强度/法术暴击系数高；双修职业（圣骑士、德鲁伊、萨满）系数居中，支持多定位。
- **"-" 表示该职业不使用该次级属性**，公式不参与计算。

##### 满级属性估算（60 级，含 295 点自由分配）

假设玩家将全部 295 点自由属性投入主属性（极端主属性构筑），则 60 级时主属性 ≈ 1级基础 + 295。示例（战士，全加力量）：

| 职业 | 1级主属性 | 60级主属性（全加主属性） | HP 估算 | PhysAtk/SpellPower 估算 |
|------|-----------|--------------------------|---------|-------------------------|
| 战士 | Str 10 | 10 + 295 = 305 | 10 + 9×4 + 60×2 = 166 | 5 + 305×0.65 ≈ 203 |
| 法师 | Int 11 | 11 + 295 = 306 | 10 + 4×2 + 60×2 = 138 | 5 + 306×0.65 ≈ 204 |
| 盗贼 | Agi 11 | 11 + 295 = 306 | 10 + 6×2.8 + 60×2 ≈ 147 | 5 + 306×0.55 ≈ 173 |

- **均衡分配**：若玩家将 295 点分散至多属性（如坦克加耐力、DPS 加主属性），实际数值介于 1 级与上述极端值之间。装备词缀会进一步放大差异。

#### 2.2.3 伤害与减伤公式

| 类型 | 公式 | 说明 |
|------|------|------|
| 原始伤害 | `rawDamage = PhysAtk/SpellPower * SkillCoeff * [1.5 if crit]` | SkillCoeff 为技能倍率，暴击时乘 1.5 |
| 护甲抵消 | `finalDamage = max(1, rawDamage - Armor)` | 1 护甲 = 抵消 1 点物理伤害，无上限，装备可叠加 |
| 抗性抵消 | `finalDamage = max(1, rawDamage - Resistance)` | 1 抗性 = 抵消 1 点魔法伤害 |
| 暴击伤害 | `CritMultiplier = 1.5` | 暴击时伤害 * 1.5，可由装备/技能修改 |

- **直接抵消**：每点护甲/抗性 = 抵消 1 点伤害，无上限，装备可叠加。

#### 2.2.4 怒气与能量规则（草案）

| 资源 | 获得方式 | 消耗示例 |
|------|----------|----------|
| 怒气 | 受到伤害: +1 怒气/2 点伤害；造成伤害: +1 怒气/4 点伤害 | 英勇打击 15 怒气，盾牌猛击 20 怒气 |
| 能量 | 固定每回合恢复 10 点 | 影袭 40 能量，背刺 60 能量 |
| 集中值 | 固定每回合恢复 6 点；稳固射击不消耗 | 奥术射击 30 集中值 |

- 具体数值在技能设计时平衡，此处为量级参考。

### 2.3 属性来源

| 来源 | 说明 |
|------|------|
| 基础属性 | 职业与等级决定的基础值 |
| 升级成长 | 每级获得固定点数，按职业主属性分配 |
| 装备词缀 | 装备提供 +Str、+Agi、+Int、+护甲、+法术抗性 等词缀 |
| Buff/Debuff | 技能或状态效果临时增减属性 |

### 2.4 属性显示与透明化

- 角色面板展示：**基础属性** + **装备加成** + **Buff 加成** = **最终属性**。
- 战斗日志与数据统计中，可展示关键衍生数值（如实际暴击率、命中率），便于玩家验证与优化构筑。

# 用户界面与用户体验 (UI/UX)

## 主界面布局

主界面采用**三栏 + 顶部栏**布局，最大化战斗信息密度：

```
┌──────────────────────────────────────────────────────────────┐
│ [Text Idle]   [当前地图名 ▼ (点击弹出地图选择)]   [用户名][退出] │  ← 顶部栏
├──────────────┬──────────────┬─────────────────────────────────┤
│  小队 (左)   │ 当前怪物(中左)│    战斗日志 (右，主区)           │
│ [英雄卡×1-5] │ [怪物卡×N]   │  [逐条动画，每条2s间隔] [暂停]   │
│              │              │  [胜/败结果 + 奖励信息]          │
│ [招募按钮]   │              │  [当前地图探索进度条]            │
└──────────────┴──────────────┴─────────────────────────────────┘
```

- **顶部栏**：游戏标题 + 当前地图（可点击弹出地图切换 Modal）+ 用户信息与退出按钮
- **左栏（小队）**：紧凑型英雄卡片，每张显示姓名、职业、等级、HP 条、资源条；点击卡片弹出详情 Modal
- **中左栏（怪物）**：当前战斗中的怪物卡片，显示名称、Tier 标签、HP 条；点击弹出详情 Modal；与小队相邻便于同时关注敌我状态
- **右栏（战斗日志）**：战斗日志逐条动画显示（每条间隔 2s），颜色区分伤害类型；提供**暂停/继续**按钮，可暂停日志滚动以便详细查看；日志跨战斗持续保留，每场战斗间以分割线分隔

## 英雄卡片（紧凑版）

| 显示内容 | 说明 |
|----------|------|
| 姓名 + 职业 | 职业文字使用对应的 WoW 职业色 |
| 等级 | Lv.N |
| HP 条 | 颜色随受伤程度变化：健康(>75%)绿、受伤(26-75%)黄、重伤(1-25%)红、死亡灰；血条与数字同色；显示当前/最大 HP |
| 资源条 | 颜色随职业变化：MP=蓝、Rage=橙、Energy=黄、Focus=青 |
| 行动中高亮 | 该英雄行动时卡片放大并带发光边框（见「行动中高亮」） |

## 怪物卡片（紧凑版）

| 显示内容 | 说明 |
|----------|------|
| 名称 + Tier 标签 | Normal/Elite/Boss 不同颜色标注 |
| HP 条 | 颜色随受伤程度变化：健康(>75%)绿、受伤(26-75%)黄、重伤(1-25%)红、死亡灰；血条与数字同色；显示当前/最大 HP |
| 行动中高亮 | 该怪物行动时卡片放大并带发光边框（见「行动中高亮」） |

### 行动中高亮

战斗过程中，正在行动的角色（英雄或怪物）卡片会短暂放大（scale 1.08）并带有**绿色发光边框**，以直观表示该角色正在执行行动。受到攻击且命中的目标会显示**红色边框**和**受伤闪烁效果**（红色背景快速淡出）；若攻击未命中则无效果。

## 战斗日志增强

### 战斗流程日志

每场战斗在日志中按以下顺序展示，各场战斗之间以分割线和换行分隔：

1. **遭遇说明**：战斗开始时显示"Your adventure party encountered XXX!"（Boss 遭遇追加 "fearsome"），以金色斜体呈现；其中怪物名称按 Tier 着色，与战斗日志中怪物姓名颜色一致（Normal/Elite/Boss）
2. **回合日志**：逐条动画显示每次行动，包含详细伤害计算过程
3. **战斗总结**：胜利/失败/平局的总结性描述，含击杀怪物数、回合数、奖励信息
4. **休息阶段**：胜利后显示休息过程，包含 HP/MP 恢复进度，直至"Rest complete"
5. **分割线**：下一场战斗开始前插入视觉分割线

### 姓名颜色

- 战斗日志中，玩家角色姓名统一使用其职业对应的 WoW 职业色（与英雄卡片一致）
- 怪物姓名按 Tier 使用不同颜色：Normal (#66aa88)、Elite (#cc88ff)、Boss (#ff6644)
- 怪物作为攻击者和被攻击者时颜色保持一致

### 伤害数字颜色

| 伤害类型 | 颜色 | 暴击表现 |
|----------|------|----------|
| 物理伤害 | 白色 (#dddddd) | 加粗 + CRIT! 标记 |
| 魔法伤害 | 蓝色 (#44aaff) | 加粗 + CRIT! 标记 |

### 战斗日志可读性

- **回合标记 [Rx]**：使用较高对比度颜色（#66aa88）与浅色背景，便于识别
- **连接词**（used、on、for）：使用 #88bb99 等可读色，避免与背景混淆
- **伤害类型**（physical）、（magic）：使用 #99ccaa 与浅色背景，与计算详情行区分

### 伤害计算详情

每条伤害日志下方附带浅绿色 (#88aa88) 计算详情行，格式为：
`ATK(raw) - Armor(defVal) = final` 或 `ATK(base) x 1.5 - Armor(defVal) = final`（暴击时）
- ATK(raw)：攻击方原始伤害
- Armor(defVal)/Resist(defVal)：防守方护甲或抗性数值
- final：最终伤害

被攻击方 HP 变化：在计算详情下方以 HP 色显示 `{被攻击者姓名} HP: before -> after/max`，便于复盘时追踪目标血量变化。

### 暴击系统

- 英雄暴击率由职业系数决定：PhysCrit = 5 + Agi * k_PhysCrit，SpellCrit = 5 + Int * k_SpellCrit
- 怪物暴击率按 Tier：Normal 5%，Elite 10%，Boss 15%
- 暴击伤害倍率：1.5x

## 角色详情面板

点击英雄卡片弹出详情 Modal，布局为左右对齐（属性名左侧、数值右侧）。角色姓名使用浅色文字 (#eeffee)，职业标签使用对应的 WoW 职业色，两者颜色区分：

## 战斗自动化

- 无任何手动战斗按钮（已移除 Explore Normal/Elite、Start Encounter、Fight Zone Boss、Recover One Turn）
- **战斗暂停**：战斗日志区域提供暂停/继续按钮，暂停时日志停止滚动、战斗动画暂停，便于玩家详细查看战斗过程
- 组件加载后自动启动战斗循环：遭遇说明 → 战斗 → 日志动画 → 战斗总结 → 自动休息（含日志） → 分割线 → 下一场战斗
- 战胜积累探索度，探索度满时自动挑战 BOSS
- 休息阶段每步间隔 2000ms，每步在日志中显示恢复进度
- 战败后同样进入恢复阶段（"Recovering from defeat..."），逐步恢复所有英雄 HP/MP 至满值后再开始下一场战斗

- **数据可视化设计**: 战斗日志语义着色、HP/资源进度条、探索度进度条

## 显示规格与适配策略

- **目标平台**: 桌面端显示器，不再进行移动端适配。
- **目标比例**: 固定按 16:9 视觉舞台进行布局，界面以该比例为基准设计与实现。
- **最低分辨率**: 1920 x 1080（1080P），低于该分辨率视为不支持运行环境。
- **布局原则**:
  - 所有核心页面以大屏信息密度为优先，不再为窄屏做断点重排。
  - 主界面采用固定舞台（16:9）+ 内容分栏布局，减少滚动依赖。
  - 在超宽或非 16:9 屏幕上，通过留边方式保持核心内容比例一致。
- **交互反馈**: 当检测到分辨率低于 1080P 时，前端展示不支持提示，并引导用户调整分辨率或更换设备。

# 技术规格 (Technical Specification)

## 技术栈总览

| 层级 | 选型 | 说明 |
|------|------|------|
| 后端 | Go | 高并发、单二进制部署、生态成熟 |
| 前端 | Vue 3 | 组件化、响应式、适合复杂数据展示 |
| 数据库 | SQLite | 初期轻量部署，使用 GORM 便于后续迁移至 PostgreSQL |
| 通信 | REST + WebSocket | REST 常规接口，WebSocket 实时推送 |

## 后端 (Backend)

- **语言与框架**: Go + Gin / Fiber
- **选型理由**:
  - 原生并发支持（goroutine），适合多玩家、战斗结算、离线收益计算等场景
  - 单二进制部署，资源占用低，适合中小规模部署
  - Web 框架成熟，便于快速开发 REST API 与 WebSocket 服务
- **关键技术**:
  - WebSocket: gorilla/websocket 或 nhooyr.io/websocket（战斗日志、聊天等实时推送）
  - 离线收益: 服务端按离线时长计算，保证防作弊与数据一致性

## 前端 (Frontend)

- **框架**: Vue 3
- **选型理由**:
  - Composition API 便于组织战斗日志、数据统计等复杂逻辑
  - 响应式系统适合实时战斗面板、挂机收益统计等动态更新
  - 组件化适合拆解主界面、战斗、角色、背包等模块
- **技术建议**:
  - 状态管理: Pinia
  - 路由: Vue Router
  - UI 风格: 复古像素风（见下方界面美术风格）
  - 显示规格: 仅支持桌面端 16:9，最低 1080P，不提供移动端响应式断点

## 数据库 (Database)

- **选型**: SQLite（MVP / Alpha 阶段）
- **适用场景**: 单服部署、玩家量级较小、读写并发不高
- **注意事项**:
  - SQLite 为单写锁，高并发写入时会排队
  - 若 Beta / 正式版预期玩家增多或需多服部署，可迁移至 PostgreSQL
- **ORM**: GORM（支持 SQLite 与 PostgreSQL，迁移时仅需调整连接配置）

## 客户端/服务端架构

- **架构模式**: 前后端分离，前端 SPA + 后端 API 服务
- **离线收益**: 服务端权威计算，客户端仅展示与上报在线/离线状态
- **实时通信**: 战斗日志、聊天等通过 WebSocket 推送；排行榜、交易等可先采用 REST + 定时刷新

## 界面美术风格

- **整体风格**: 黑底绿字怀旧电脑风格（CRT 终端风格），与「复古极简美学」设计理念一致
- **实现要点**:
  - **配色方案**: 
    - 背景：纯黑色 (#000000)
    - 主文字：绿色 (#00ff88)，营造 CRT 显示器效果
    - 边框：绿色，使用 1-2px 实线
    - 强调色：亮绿色 (#00ffaa) 用于高亮和交互元素
    - 错误/警告：红色 (#ff6666) 用于错误，橙色 (#ffcc66) 用于警告
    - **语义色扩展**（用于战斗信息差异化显示）：
      - HP 条与数字：颜色随受伤程度变化，健康(>75%)绿(#44ff88)、受伤(26-75%)黄(#ffdd66)、重伤(1-25%)红(#ff4444)、死亡灰(#888888)
      - MP 条：蓝色 (#44aaff)
      - 物理伤害：橙色 (#ffaa44)
      - 魔法伤害：紫色 (#cc88ff)
      - 技能标注：黄色 (#ffee66)
      - 胜利结果：青色 (#00ffcc)
      - 失败结果：红色 (#ff4444)
      - 金币：金黄色 (#ffcc44)
      - 经验：浅绿色 (#88ffaa)
      - Boss：橙红色 (#ff6644)
  - **滚动条**: 自定义样式匹配暗绿终端主题（6px 窄滚动条，暗色轨道 #0a0a0a，绿色滑块 #1a3a1a，悬浮 #2a5a2a）
  - **字体**: 像素字体（如 Press Start 2P、VT323、Silkscreen）或等宽字体，强化终端感
  - **视觉效果**: 
    - 可添加轻微的文字发光效果（text-shadow）模拟 CRT 荧光
    - 边框使用简洁的实线，避免过度装饰
    - 保持简洁的布局，强化「命令行终端」的怀旧感
  - **图像**: `image-rendering: pixelated` 或 `crisp-edges` 保持像素图不模糊
  - **布局**: 网格化、固定宽高，强化「格子感」和「终端窗口」感

## 防作弊策略

- **核心原则**: 所有关键数值在服务端计算，客户端仅负责展示与输入
- **具体措施**:
  - 战斗结算、装备掉落、离线收益等逻辑全部在服务端执行
  - 客户端不参与任何权威数值计算，仅接收服务端结果并展示
