# 装备系统设计详细说明

## 一、设计思路概述

装备系统借鉴《暗黑破坏神2》的经典设计，防具参考《流放之路》的护甲/抗性双属性体系。核心围绕**装备部位与类型**、**防具防御属性**、**品质与词缀**、**层级与掉落**、**属性要求与随机性**五个维度展开。

## 二、装备部位与类型 (Equipment Slots & Types)

### 2.1 武器槽位

| 槽位 | 英文标识 | 可装备类型 | 说明 |
|------|----------|------------|------|
| 主手 | MainHand | 单手武器 | 可与副手组合使用 |
| 副手 | OffHand | 盾牌、法术宝珠（Orb） | 持主手单手武器时可装备；双手武器占用主手+副手 |
| 双手 | TwoHand | 双手武器 | 装备于主手槽位，占用主手与副手；副手显示「—」不可装备，UI 不单独显示 TwoHand 槽位 |

- **主手 + 副手**：主手装备单手武器时，副手可装备盾牌（物理/坦克向）或法术宝珠 Orb（法术向，被动持握，仅提供属性加成，不造成武器伤害）。
- **单手杖**（魔杖/权杖）属于**主手武器**，可造成法术伤害，与盾牌/宝珠组合使用；不可与法术宝珠混淆。
- **双手武器**：装备双手武器时，主手与副手均被占用，无法再装备盾牌或法系副手。
- **武器伤害范围**：物理/法术武器的 PhysAtk+/SpellPower+ 为**范围**（如 3–5、9–14），**每次攻击时**在范围内随机，而非掉落时 roll 一次；详见[04-classes-attributes.md](./04-classes-attributes.md)2.2.3.1。

### 2.3 防具防御属性与属性要求（参考流放之路）

防具（头盔、盔甲、手套、鞋子、腰带）具有**物理护甲**与**法术抗性**两个防御属性：

| 防御属性 | 英文标识 | 效果 |
|----------|----------|------|
| 物理护甲 | Armor | 减免所受物理伤害 |
| 法术抗性 | Resistance | 减免所受魔法伤害 |

**护甲与抗性分配规则**：

- **保底**：每件防具至少提供 **1 点护甲** 和 **1 点抗性**
- **总值**：护甲 + 抗性的总和由**底材等级与层级**决定，不同级别和品阶总值不同
- **随机分配**：护甲与抗性无固定侧重，**掉落时**在满足「各≥1」的前提下随机分配（如总值为 10，则可能 2 护甲 + 8 抗性，或 6 护甲 + 4 抗性等）

**属性要求**：随底材层级与底材族而定（见 4.4 底材基础属性表），与护甲/抗性随机分配无关。

**随机分配实现**：掉落时先 roll 总值（在底材范围内），再在 [1, 总值−1] 间随机取护甲，抗性 = 总值 − 护甲，保证二者各≥1。

## 三、品质分级与词缀数量 (Quality & Affix Count)

| 品质 | 英文标识 | 词缀数量 | 说明 |
|------|----------|----------|------|
| 普通 | Normal | 0 | 白装，无词缀，仅基础属性 |
| 魔法 | Magic (Blue) | 1–2 个词缀 | 蓝装，随机 1 或 2 个词缀；**词缀随机范围更大** |
| 稀有 | Rare (Yellow) | 3–5 个词缀 | 黄装，随机 3–5 个词缀；靠词缀条数与组合取胜；单条词缀随机规则与蓝装一致（见下） |
| 暗金 | Unique | 固定属性 + 特殊效果 | 固定词缀组合，具有独特特效 |

- **词缀来源**：魔法/稀有装备从词缀池中随机抽取前缀、后缀；暗金装备拥有预设的固定属性与特殊效果。
- **蓝装 vs 黄装**：单条词缀的随机区间**相同**（均对词缀池中的基准 min/max 应用 0.7–1.3 倍换算，见 7.1）。差异主要在**词缀条数**：蓝装 1–2 条，黄装 3–5 条；黄装通常总属性更高，但单条仍可能出现「非满 roll」，蓝装仍可能因单条极品而有竞争力。
- **透明化**：玩家获得装备后，可查看该装备各属性的**随机范围**，判断是否为「极品」 roll。

## 四、装备层级与怪物掉落 (Item Tier & Drop)

### 4.1 装备层级（类似暗黑2）

| 层级 | 英文标识 | 说明 | 掉落条件 |
|------|----------|------|----------|
| 普通 | Normal | 基础层级，属性与词缀数值较低 | 任意等级怪物 |
| 拓展 | Exceptional | 中级层级，同词缀加成更高 | 怪物等级超过阈值后开始掉落 |
| 精英 | Elite | 最高层级，同词缀加成最高 | 怪物等级超过更高阈值后开始掉落 |

- **层级与词缀**：同一词缀在不同层级装备上，数值范围不同；精英 > 拓展 > 普通。
- **怪物等级阈值**：具体数值可配置，如怪物等级 ≥ 20 可掉拓展，≥ 40 可掉精英。

### 4.2 词缀层级

词缀本身也分为三个级别，与装备层级对应：

| 词缀级别 | 说明 | 数值范围 |
|----------|------|----------|
| 普通级 | 低等级词缀 | 加成较小 |
| 拓展级 | 中级词缀 | 加成中等 |
| 精英级 | 高级词缀 | 加成最大 |

- **对应关系**：普通装备仅能出现普通级词缀；拓展装备可出现普通级与拓展级；精英装备可出现全部三个级别。
- **同类词缀**：如「+力量」，普通级 +1~3，拓展级 +4~8，精英级 +9~15（数值为示例，可配置）。

### 4.3 底材名称表 (Base Item Names)

每个部位有 **6 个底材族（F1–F6）**，每族包含 **普通 / 拓展 / 精英** 三个层级，共 **18 个底材 / 部位**。

- **底材族 F1–F6**：代表同一类型装备在不同力量层次的形态；F1 最基础，F6 最强大。
- **层级分布**：同族内，**普通**对应 Lv 1–20，**拓展**对应 Lv 21–40，**精英**对应 Lv 41–60；三件装备分别在游戏早、中、晚期掉落，等级要求与属性加成逐层显著提升。
- **命名规律**：同族内 普通 → 拓展 → 精英 听起来越来越强悍；F1 → F6 各族整体由朴素递进至史诗。

### 4.4 底材基础属性表 (Base Item Stats)

每个底材按**层级**分三组列出，各层级有独立的等级要求区间，属性要求和加成随层级显著提升。数值为白板基础范围（未附词缀），词缀额外叠加。

**设计规律：**
- **防具**：护甲与抗性各≥1，**护甲+抗性总值**随层级与底材族递增；分配比例**随机**，无固定侧重
- **盾牌**三层级均提供护甲+格挡率，全程需力量，强度随层级递增
- **2H 武器**伤害约为同族同层级 1H 的 1.5 倍，属性要求更高
- **匕首与弓**需敏捷，法系武器需智力，其余物理武器需力量
- 同层级内 F1→F6 等级要求与加成按约 4 级递增
- **相邻族（如 F1 与 F2）**：普通级相邻底材的数值区间需拉开，避免 Lv1 与 Lv4 白装在体感上几乎无差别（实现数据见 `frontend/src/data/itemBases.js`）。

## 五、装备要求与属性需求 (Requirements)

| 要求类型 | 说明 |

| 等级要求 | 角色等级需达到装备所需等级才能穿戴 |
| 属性要求 | 力量、敏捷、智力、精神等需满足装备要求；防具属性要求随底材层级而定（见 4.4 底材表），与护甲/抗性随机分配无关 |

## 六、词缀随机范围与可见性 (Affix Roll & Visibility)

### 6.1 随机范围

- 词缀属性在**预设范围内随机**生成，类似暗黑2。
- **品质差异**：蓝装与黄装对**同一词缀家族、同一装备层级**使用**同一套单条随机区间**（由基准范围 ×0.7~×1.3 导出）。黄装优势主要来自**更多词缀条数**（3–5 对 1–2），而非更窄的单条区间。
- 例如：同一家族护甲前缀下，蓝装可能出现一条高 roll，黄装则叠加多条词缀；总强度比较需看条数与每条 roll 的综合结果。

### 6.2 可见性

| 时机 | 可见内容 |
|------|----------|
| 获得装备后 | 显示该装备**各属性的随机范围**（如 +5~12 护甲），以及当前 roll 出的实际数值 |
| 装备对比 | 对比两件同类装备时，可看到各自的 roll 值与范围，判断是否为「极品」 |

- **设计意图**：增加「打到好装备」的惊喜感与成就感；玩家可明确判断「这件装备是否值得保留」。

### 6.3 装备命名规则 (Item Naming)

#### 6.3.1 命名格式

| 品质 | 命名格式 | 示例 |
|------|----------|------|
| 普通 (白) | **底材名** | 便帽、皮甲、短弓 |
| 魔法 (蓝) | **前缀 + 底材**；或 **底材 + · + 后缀**；双词缀为 **前缀 + 底材 + · + 后缀** | 坚固便帽、便帽·熊之、坚固便帽·熊之 |
| 稀有 (黄) | **主前缀 + 底材 + · + 主后缀 + ，+ 称号**（主词缀规则同下） | 强力王冠·泰坦之，老兵 |
| 暗金 (金) | **固定名称**（预设，不随词缀变化） | 见暗金装备表（待扩展） |

#### 6.3.2 蓝装命名细则

- **1 前缀 0 后缀**：`前缀名` 与 `底材名` 直接相连，如 `坚固便帽`、`强力长剑`
- **0 前缀 1 后缀**：`底材名` + `·` + `后缀名`，如 `便帽·熊之`、`长剑·打击之`
- **1 前缀 1 后缀**：`前缀名` + `底材名` + `·` + `后缀名`，如 `坚固便帽·熊之`

#### 6.3.3 黄装命名细则

黄装有 3–5 条词缀，无法全部放入名称。采用**主词缀 + 称号**方式：

1. **主词缀选取**：从所有前缀中取**词缀层级最高**的一条作为主前缀；从所有后缀中取**词缀层级最高**的一条作为主后缀（层级：精英 > 拓展 > 普通；同层级时按词缀池顺序或随机）。
2. **主名格式**：与蓝装双词缀相同，`主前缀 + 底材 + · + 主后缀`。
3. **稀有称号**：在主名后追加中文逗号 `，` 与**称号**（实现中无英文 "the"）。称号从预设池中**根据词缀数量或随机**选取，用于标识「多词缀稀有装」，与蓝装形成视觉区分。
4. **称号池示例**（可配置）：老兵、冠军、荣光、灾厄、恩惠、守御、贤者、风暴、烈焰、寒霜 等；每件黄装生成时随机或按规则选取一个。

**示例**：
- 蓝装：`坚固王冠·熊之`（1 前 1 后）
- 黄装：`强力王冠·泰坦之，老兵`（2 前 2 后，主词缀为 强力 + 泰坦之，称号 老兵）

- **详情展示**：装备详情 Modal 中列出全部 3–5 条词缀及 roll 值，名称仅作概览与辨识。

## 七、词缀池设计 (Affix Pool)

> 本章以**全职业覆盖**为目标维护词缀体系；词缀与部位的对应关系按规则本身定义（例如物理武器专有词缀绑定**物理攻击武器**，而非特定职业专属）。后续新增职业时，在通用池与各部位专用池上按底材与职业需求扩展即可。

### 7.1 词缀体系概述

| 维度 | 规则 |
|------|------|
| **类型** | 前缀 (Prefix) / 后缀 (Suffix)；每件装备最多各 1 个前缀、1 个后缀 |
| **品质上限** | 魔法（蓝）最多 2 条词缀（1 前 + 1 后）；稀有（黄）3–5 条（最多 3 前 + 3 后） |
| **词缀层级** | 普通（掉落自 Lv 1–20 怪物）/ 拓展（Lv 21–40）/ 精英（Lv 41–60） |
| **分类** | **通用词缀**：可出现在所有部位；**专用词缀**：仅出现在特定 1–3 个部位 |

**蓝装 vs 黄装词缀范围规则：**

| 装备品质 | 随机范围规则 | 说明 |
|----------|------------|------|
| 蓝（魔法）| 下限 = max(1, floor(基准×0.7))，上限 = ceil(基准×1.3) | 词缀值至少 +1，不出现 +0 |
| 黄（稀有）| **与蓝装相同**（同一公式作用于词缀池中的基准 min/max） | 与蓝的差异主要在词缀条数（3–5），而非单条区间 |

> 词缀池中的 min/max 为**设计基准**；蓝装与黄装均按上式换算为玩家可见的 roll 区间。

> 说明：`armorPct` / `resistancePct` / `maxHp` 已移至 7.3 防具专用候选；其中百分比词缀按单件防具自身数值放大，不作用于角色全身总护甲/总抗性。
> 说明：`hitPct` / `dodgePct` 已从本节通用候选迁至 7.3（命中归武器专用、闪避归防具专用）；两者仅提供面板加成，实际命中/闪避对抗结算规则以 [03-combat.md](./03-combat.md) 为准。
> 说明：当前命中/闪避对抗为统一结算口径，不区分物理命中/法术命中或物理闪避/法术闪避。
> 说明：暴击相关候选已拆分为物理与法术两轨（`physCritPct` / `physCritDmgPct` / `spellCritPct` / `spellCritDmgPct`），与当前角色次级属性与战斗结算口径保持一致。
> 说明：`magicFindPct`（MF）仅作用于掉落品质权重（普通/魔法/稀有/暗金等），不影响金币掉落；与 `goldFindPct` 分轨。

**筛选建议（先选 4-6 族进入首批扩充）**

- 防御向通用候选优先保留 `hpRegen`；`maxHp` 与百分比防御放到防具专用池处理（见 7.3）。
- 再补一个**资源向通用词缀**：`manaRegen` 或 `hpRegen`，避免仅靠武器专有实现续航。
- 输出向建议按职业主轴优先：物理向先上 `physCritPct` + `physCritDmgPct`，法术向先上 `spellCritPct` + `spellCritDmgPct`；命中词缀由武器专用池承接（见 7.3）。
- `goldFindPct` 建议最后进入，避免过早影响经济平衡。
- `magicFindPct` 建议与 `goldFindPct` 错峰投放（先低档后高档），避免经济与掉落品质同时膨胀。
- 若首批实装后金币曲线仍偏缓，可优先在不改触发规则的前提下继续微调 `goldFindPct` 数值区间。

##### 机制说明（物理武器专有，回合制）

**1）触发范围**

- 仅在该英雄**本回合行动中**，使用**受武器物理攻击面加成的伤害**时生效：含**普通攻击**、明确按武器物攻结算的战士技能（如英勇打击、顺劈等，以技能定义为准）。
- **不触发**：纯法术伤害、不受武器 `physAtk` 加成的技能、治疗、无伤害的嘲讽等。

**2）生命偷取（Life Steal，%）**

- 在**本段物理伤害**对目标 HP 扣减结算完成后，治疗攻击者：`治疗量 = floor(本次攻击最终物理伤害 × 偷取%)`，**至少为 0**；若未命中、伤害为 0 或被完全闪避，则不回复。
- 类比 WoW：**按「打出去的那一下」物理伤害**吸血，而非按面板攻击；与「命中回血」可叠乘存在（先判伤害，再分别加两段治疗）。

**3）命中回血（Life on Hit，固定值）**

- 条件：本段物理伤害 **> 0**。回复量 = 词缀 roll 的固定值（不受伤害高低放大，便于前期武器也有感）。
- 与生命偷取**独立**：同一击可同时触发两者。

**4）附加魔法伤害（Weapon-added Magic Damage）**

- 类比附魔 **火焰/冰霜伤害**、D2 元素伤害词条：在同一次攻击、同一目标上，在**物理段结算之后**，再追加一段 **魔法类型** 伤害，数值在词缀区间内随机取整。
- **减伤**：仅使用目标的 **法术抗性**（Resistance）参与魔法伤害公式，**不经物理护甲**；与物理段分开展示在战斗日志（或合并一行分两段公式，由 UI 约定）。
- **不吃**物理暴击的暴伤放大（可单独配置「附加魔伤是否允许法术暴击」；默认否，以简化）。

**5）护甲穿透（固定）与无视护甲（%）**

- **固定穿透**：`有效护甲 = max(0, 目标当前护甲 − 穿透值)`，再代入现有物理减伤公式（见 [04-classes-attributes.md](./04-classes-attributes.md)）。
- **百分比无视**：`有效护甲 = floor(目标当前护甲 × (1 − 无视%))`，再代入减伤。
- **与技能破甲（Sunder 等）**：建议 **先** 用 debuff 修正目标护甲，**再** 应用武器穿透与无视（顺序以程序统一为准，文档与日志需写清）。
- **叠加上限**：固定穿透 + 百分比无视 + debuff 不得使有效护甲为负；若需防秒杀，可对「无视%」设软上限。

**6）推荐结算顺序（单目标、单次物理技能/普攻）**

1. 取武器与 buff 后的物理基础伤害 → 暴击判定 → 得到「原始物理伤害」候选。  
2. 应用 **固定穿透** → **百分比无视** → **护甲减伤** → 得到「最终物理伤害」，扣血、计仇恨。  
3. 结算 **生命偷取**、**命中回血**。  
4. 若有 **附加魔法伤害**，单独 roll 魔法段 → 抗性减伤 → 扣血、计仇恨（若设计为无仇恨则注明）。  
5. 战斗日志按上述顺序输出，便于玩家复盘。

**战斗日志（实现约定）**

- **主段公式行**：`damageFormulaEquation` 使用 `targetDefense` 作为「护甲/抗性抵消」括号内的数值，该值为 **穿透与无视百分比之后** 的有效防御；若存在武器多段伤害，主段右侧使用 `primaryFinalDamage`（主物理/主法术段），与合计总伤 `finalDamage` 区分。
- **武器机制明细行**：`weaponMechanicLines`（主界面战斗日志详情区）追加中文说明行，包括：穿透/无视含义提示、合计对生命伤害、附加魔法伤害、奥术追伤、生命偷取与命中回血、魔力回流与施法回蓝；**普攻**后若因武器回蓝改变了法力，可显示当前法力（技能施法条目的法力仍以 `manaAfter` 为准）。
- **格挡（物理）**：英雄成功格挡物理攻击时，明细行追加「格挡成功」；若格挡伤害减免（`blockDrPct`）使有效伤害低于格挡前，则追加「格挡减伤后有效伤害 *X*」（*X* 为护盾/扣血结算前的有效伤害，与主公式行右侧在格挡后的数值一致）；若有格挡反击，追加「格挡反击：对 *攻击者* 造成 *Y* 点物理伤害」。主公式行在格挡命中时，等号右侧为格挡前有效物理伤害（`physicalDamageBeforeBlock`），护盾吸收句仍按格挡后的 `finalDamage` 计算净生命损失。
- **治疗文案**：带伤害的战士技能中，`healFromSkill`（如嗜血）与武器吸血在日志中分开展示，避免混为一笔。

**7）平衡与扩展**

- 附加魔法伤害偏高时易碾压高甲怪：可通过**同层数值略低于**纯物攻%词缀、或提高精英怪抗性作环境制约。
- 后续若增加「对亡灵/恶魔增伤」等，另开词缀族，不与上表混层。

#### 法术攻击武器专有词缀（主手魔杖/权杖、双手法杖）

> **设计取向**：**不必**与物理武器词缀一一对应；法术侧突出 **法力循环** 与 **纯魔法输出结构**。**不采用**需跨回合单独结算的武器 DoT（持续伤仍由**技能**提供）。单次施法内可结算完毕。借鉴施法节奏、降抗、多段魔法打击等思路；**不**使用「法术吸血生命」「硬拗物理副伤」等与法系身份违和的包装。

##### 机制说明（法术武器专有，回合制）

**1）触发范围**

- 仅在该英雄**本回合行动中**，使用**受武器法术强度加成的伤害性法术**时生效（如寒冰箭、火球、神圣伤害技能等，以技能定义是否吃 `spellPower` 为准）。
- **不触发**：纯治疗、套盾、无伤害增益、嘲讽、物理伤害技能（除非技能明确为物法混合且法术段单独结算——以技能定义为准）。

**2）魔力回流（Mana Reflux / %）**

- **不是**生命偷取：仅将**法力**返还施法者。在**最终魔法伤害**对目标 HP 扣减结算完成后，返还法力 `floor(最终魔法伤害 × 回流%)`，**至少为 0**；未命中、伤害为 0 或免疫则不返还。
- 若技能为多段伤害，默认按「**技能一次行动**」对**总魔法伤害**合并结算一次回流（或按段分摊，需在技能表注明）。
- 与物理武器的「生命偷取」**无对应关系**——法系词缀专注**资源**，不混用「吸血」叙事。

**3）施法回蓝（固定值）**

- 条件：本施法造成 **>0** 的魔法伤害。回复量 = 词缀 roll 的固定值（**不**按伤害高低放大，便于低等级武器也有感）。
- 与**魔力回流**独立、可叠加；与装备、精神、回合回蓝**可叠加**（见 [04-classes-attributes.md](./04-classes-attributes.md) 法力相关）。

**4）奥术追伤（Arcane Follow-up Damage）**

- **不是**「为了对偶物理武器而硬加一段物理」：本段仍为 **魔法伤害**，走**抗性**与**法术穿透**（与主段共用同一套有效抗性，或主段结算后再对追伤段套用穿透——**须程序统一**）。
- 用途：在同一次施法中形成**第二段魔法**（如主技能 + 武器迸发的奥术溅射），便于做爆发、叠日志，而不引入护甲/物伤公式。
- **暴击**：可与主段独立 roll（避免双段必暴），或追伤固定不暴击，以降低复杂度。

**5）法术穿透（固定）与无视抗性（%）**

- **固定穿透**：`有效抗性 = max(0, 目标当前抗性 − 穿透值)`，再代入现有魔法减伤公式（见 [04-classes-attributes.md](./04-classes-attributes.md)）。
- **百分比无视**：`有效抗性 = floor(目标当前抗性 × (1 − 无视%))`。
- **与技能减抗**（如虚弱诅咒、法术易伤 debuff）：建议 **先** debuff，**再** 武器穿透与无视（程序统一顺序后文档与日志一致）。
- **叠加上限**：有效抗性不得为负；若环境需要，可对「无视%」设软上限。

**6）推荐结算顺序（单目标、单次法术技能）**

1. 取法强与 buff 后的魔法基础伤害 → 法术暴击判定 → 得到「原始魔法伤害」候选。  
2. 应用 **固定法术穿透** → **百分比无视抗性** → **抗性减伤** → 得到「最终魔法伤害」，扣血、计仇恨。  
3. 结算 **魔力回流**（按最终魔伤转法力）、**施法回蓝**（固定）。  
4. 若有 **奥术追伤**，单独 roll 追伤段 → 共用或重复穿透/抗性流程 → 扣血、计仇恨。  
5. 战斗日志按上述顺序输出，便于复盘。

**7）与物理武器专有的关系（非强制对称）**

| 维度 | 物理武器专有 | 法术武器专有（本节） |
|------|----------------|----------------------|
| 生存向 | 生命偷取、命中回血 | **不**以武器词缀为主推吸血；可选通用/防具解决生存 |
| 资源向 | （无强绑定） | **魔力回流**、**施法回蓝** |
| 额外伤害段 | 附加**魔法**伤害（走抗性） | **奥术追伤**（仍为魔法，走抗性；非物理副伤） |
| 穿透 | 护甲固定穿透、无视护甲% | 抗性固定穿透、无视抗性% |

**8）平衡与扩展**

- 法术穿透与无视% 过强时易碾压高抗怪：可与怪物「抗性硬底」、或 PvE 系数挂钩。
- 火焰/冰霜等 **DoT 与持续燃烧** 仍由**技能与天赋**提供，武器词缀不重复造持续伤，避免与回合日志规则冲突。

### 7.4 词缀互斥规则

| 规则 | 说明 |
|------|------|
| 同前缀只能一条 | 同件装备不能同时拥有两个前缀（如不能同时有 Sturdy 和 Reinforced） |
| 同后缀只能一条 | 同件装备不能同时拥有两个后缀 |
| 同效果不同来源可叠 | 盾牌底材格挡率 + 词缀格挡率正常叠加；护甲来自装备底材 + 词缀正常叠加 |
| 武器附加魔伤 / 反伤不互斥 | 武器的附加魔法伤害与防具的反伤分属不同装备与不同伤害类型，可同时存在于角色身上 |
| 物理 / 法术武器词缀 | 仅 `physAtk` 的武器可 roll 物理武器专有词缀；仅 `spellPower` 的武器可 roll 法术武器专有词缀；底材类型不符则整族不可选（**实现专用池后**生效） |

## 八、掉落率与节奏控制 (Drop Rate)

| 设计原则 | 说明 |
|----------|------|
| **低掉率** | 装备整体保持较低掉落率，减少垃圾装备数量 |
| **节约时间** | 玩家无需频繁处理大量无用装备，降低背包管理负担 |
| **惊喜感** | 每次掉落装备时，期待感与成就感更强 |

- **可配置**：具体掉落概率可通过配置调节，便于平衡迭代。（实现：`frontend/src/game/equipment.js` 中每场胜利按怪物独立掷骰；普通遭遇基础概率当前约 **8%** 每怪，精英/首领为倍数加成。）
- **与核心循环衔接**：对应「优化 (Optimize)」—— 装备稀缺使每次获得都更有价值，强化构筑与策略的深度。

### 8.1 成色与寻宝值 (Quality & Magic Find)

**核心原则：装备成色与所在区域无关，仅由寻宝值（Magic Find，简称 MF）决定。**

- **寻宝值（MF）**：角色或队伍的一项隐性/显性属性，可通过装备词缀、技能、Buff 等方式提升；战斗掉落结算按**小队平均 MF**生效（非求和）。
- **成色独立于区域**：即便在等级最低的区域（如 Lv 1 地图），理论上也存在极小概率掉落蓝装、黄装乃至暗金装备；高等级区域不会「锁定」成色，低等级区域也不会「禁止」高品质产出。
- **惊喜感设计**：玩家在练号或低级刷图时偶然获得一件黄装甚至暗金，是系统刻意保留的「彩票时刻」，强化长期留存动力。

| MF 影响维度 | 说明 |
|-------------|------|
| 蓝装（Magic）概率 | 小队平均 MF 提升白装升级为蓝装的概率 |
| 黄装（Rare）概率 | 小队平均 MF 提升蓝装升级为黄装的概率（层叠计算） |
| 暗金（Unique）概率 | MF 对暗金的提升效果按折扣系数计算，收益递减（后续拓展） |
| 装备层级 | **不受 MF 影响**；装备层级仍由怪物等级决定（见 4.1） |

> **设计说明**：装备层级（普通/拓展/精英底材）依然与区域挂钩，以保证高等级区域的核心价值；但装备「成几色」完全由 MF 控制，两个维度正交，互不干扰。这样低级玩家在早期区域也有动力刷 MF 词缀、追求高品质装备，而高级区域的吸引力则来自更高层级底材与词缀数值上限。

### 8.2 怪物层级对掉落的影响 (Monster Tier Drop Modifier)

怪物层级（普通 / 精英 / 首领）影响装备掉落概率与成色分布，与 MF 叠加生效：

| 怪物层级 | 装备掉落概率 | 成色分布（相对普通怪） | 特殊规则 |
|----------|--------------|------------------------|----------|
| 普通 (Normal) | 基准概率 | 基准 | 无 |
| 精英 (Elite) | **更高** | **蓝装 / 黄装概率更高** | 无 |
| 首领 (Boss) | **最高** | **蓝装 / 黄装 / 暗金概率更高** | **必然掉落 ≥ 1 件品质至少为魔法（蓝）的装备** |

- **掉落概率**：精英怪的装备掉落概率高于普通怪；Boss 的装备掉落概率最高。具体倍数可配置。
- **成色加成**：在 MF 基础上，精英与 Boss 的成色 roll 有额外加成——同场战斗中出现蓝装、黄装乃至暗金的概率更高。
- **Boss 保底**：击败 Boss 时，**至少掉落 1 件**品质为魔法（蓝）或以上的装备；若 Boss 本身掉落多件，则其中至少一件为蓝装及以上；若 Boss 按概率只应掉 0 件，则强制补发 1 件蓝装。
- **与 MF 叠加**：怪物层级提供的加成与玩家 MF 乘算，两者共同决定最终掉落结果（其中 MF 为小队平均值）。

> 面板约定：角色详情「词缀加成」中的 `goldFindPct` / `magicFindPct` 提示需显示**当前英雄值**与**小队平均值**，并明确战利品结算按小队平均值生效。
- **戒指与项链成色下限**：戒指、项链无底材基础属性，白色成色无任何加成；因此**仅掉落蓝色及以上成色**，若 roll 出白色则强制提升为蓝色。

## 九、与核心循环的衔接

| 循环阶段 | 装备系统贡献 |
|----------|--------------|
| **组建 (Assemble)** | 招募角色时考虑其职业与装备类型限制 |
| **放置 (Engage)** | 战斗胜利后掉落装备，怪物等级影响装备层级与词缀级别 |
| **分析 (Analyze)** | 通过装备词缀与 roll 值评估角色构筑强度 |
| **优化 (Optimize)** | 更换装备、追求极品 roll、根据属性与构筑选择护甲/抗性偏向的防具 |

# 背包系统设计详细说明

## 一、设计思路概述

背包系统为简化设计，核心目标是让玩家能够管理战斗掉落的装备与物品，同时通过「出售」机制提供清理背包的途径，降低管理负担。

## 二、背包基础规格

| 属性 | 数值 | 说明 |
|------|------|------|
| 栏位数量 | 100 | 每个玩家账户固定拥有 100 个背包栏位 |
| 栏位类型 | 统一格式 | 每个栏位存放一件装备或物品（不区分类型，不堆叠） |
| 扩展 | 预留付费扩展 | 初期固定 100 格；后续可通过付费解锁更多栏位 |

## 三、物品获取与背包满载规则

- **获取来源**：战斗胜利掉落；商店购买（见「商店系统设计详细说明」）；后续可能包含任务奖励等（待扩展）。
- **背包满载**：当背包已有 100 件物品时，新掉落的装备与物品**无法进入背包**，直接丢弃。
- **满载提示**：背包满时，战斗日志或顶部栏显示提示（如 "Inventory full — loot discarded!"），提醒玩家及时清理。
- **战斗不受影响**：背包满不影响战斗流程、经验与金币的正常获取；仅装备掉落被跳过。

## 四、出售物品

| 规则 | 说明 |
|------|------|
| 操作方式 | 玩家在背包界面点击物品，可选择「出售」 |
| 出售结果 | 物品从背包移除，获得对应金币；金币立即计入玩家账户 |
| 出售价格 | 由物品品质、层级与部位共同决定：品质基础价（普通 8 / 魔法 25 / 稀有 60 / 暗金 150）× 层级系数（普通 1 / 拓展 2 / 精英 4）× 部位系数（主手/双手 1.5、副手 1.3、护甲 1.2、头盔 1.1、护腕/靴子 1、腰带 0.9、项链/戒指 1.1）；可配置 |
| 批量出售 | 暂不支持，每次出售单件（后续可扩展） |
