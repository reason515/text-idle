# 仇恨系统设计 (Threat / Aggro)

> 怪物每回合根据仇恨值选择攻击目标；玩家通过伤害、治疗、嘲讽等技能影响仇恨分布。与策略配置（[10-tactics.md](./10-tactics.md)）中的 `ally-ot`、`highest-threat`、`tank` 等规则衔接。

## 一、设计目标与约束

| 目标 | 说明 |
|------|------|
| **怪物目标选择** | 怪物每回合攻击仇恨值最高的英雄（受嘲讽等强制效果覆盖） |
| **坦克定位** | 防护战士等通过高仇恨技能、嘲讽维持仇恨，保护队友 |
| **策略可配置** | 支持 `ally-ot`（OT 检测）、`highest-threat`（目标规则）、`tank`（友方目标） |
| **透明化** | 战斗日志与 UI 展示仇恨变化（OT 事件、目标原因、仇恨数值等），便于战后复盘与策略优化；详见 6.2 |

**约束**：回合制、每回合一次行动、无实时操控；仇恨在每回合行动时结算。

---

## 二、核心概念

### 2.1 仇恨表 (Threat Table)

| 概念 | 说明 |
|------|------|
| **每怪独立** | 每只怪物维护独立的仇恨表，记录对每位存活英雄的仇恨值 |
| **非负整数** | 仇恨值为非负整数，初始为 0 |
| **仅增不减（基础版）** | MVP 阶段仇恨不随时间衰减，仅通过技能与行动增减 |

### 2.2 怪物目标选择

怪物每回合行动时，按以下顺序决定攻击目标：

| 优先级 | 条件 | 目标 |
|--------|------|------|
| 1 | 处于**嘲讽/强制攻击**状态 | 攻击施加该效果者（Taunt / Challenging Shout） |
| 2 | 无强制效果 | 攻击仇恨表中仇恨值**最高**的英雄 |
| 3 | 仇恨并列最高 | 随机选择其一 |

### 2.3 坦克 (Tank) 定义

在策略系统中，「坦克」由**玩家在小队面板中指定**（勾选「坦克」的英雄，每队最多一名）：

| 场景 | 定义 |
|------|------|
| **指定坦克** | 玩家在小队面板勾选 `isTank` 的英雄 |
| **未指定时** | 仇恨/坦克相关战术选项不可用；`ally-ot`、`targetRule: tank` 等依赖指定坦克 |
| **指定坦克存活** | `getTank` 返回指定坦克；`ally-ot` 判定「怪物最高仇恨目标 ≠ 指定坦克」 |
| **指定坦克阵亡** | 回退至威胁表：对多数怪物仇恨最高的英雄 |

## 三、仇恨生成 (Threat Generation)

### 3.1 伤害产生的仇恨

| 行为 | 仇恨公式 | 说明 |
|------|----------|------|
| **普攻造成伤害** | `threat = finalDamage` | 1 伤害 = 1 仇恨（仅对受击怪物） |
| **技能造成伤害** | `threat = finalDamage * threatMultiplier` | 见技能配置 |
| **破甲 (Sunder Armor)** | `threat = round((finalDamage + sunderArmorReduction) * 1.5)` | `sunderArmorReduction` 为施放后目标身上破甲减甲总值（含叠层）；先与伤害相加为基数，再乘高仇恨倍率 1.5 |
| **未命中/闪避** | `threat = 0` | 不产生仇恨 |

- **目标**：仇恨仅施加于**被攻击的怪物**，不扩散至其他怪物。
- **AOE 伤害**：对每个命中的怪物分别计算仇恨，各自施加于对应怪物。

### 3.2 技能仇恨倍率 (Threat Multiplier)

| 倍率档位 | 数值 | 适用技能 |
|----------|------|----------|
| 标准 | 1.0 | 普攻、英勇打击、嗜血、顺劈斩等 |
| 高仇恨 | 1.5 | 破甲、复仇、嘲讽（施放时附加）、盾牌猛击 |
| 额外仇恨 | 1.3 | 盾牌猛击（在 1.2x 伤害基础上再乘 1.3 仇恨） |

- **配置方式**：技能定义中增加 `threatMultiplier` 字段，默认 1.0。
- **治疗产生仇恨**：治疗对**所有存活怪物**产生仇恨，公式 `threat = healAmount * 0.5`（可配置），避免治疗轻易 OT。
- **套盾产生仇恨**：真言术：盾等护盾类技能对**所有存活怪物**产生**低仇恨**，公式 `threat = absorbAmount * 0.25`（治疗倍率的一半），因护盾为预判性防护、未直接恢复生命。

### 3.3 嘲讽与强制攻击

| 技能 | 效果 | 仇恨处理 |
|------|------|----------|
| **嘲讽 Taunt** | 目标怪物接下来 2 次行动强制攻击施放者 | 施放时立即将该怪物对施放者的仇恨设为**当前最高仇恨 + 10%**（或固定值），确保后续不因伤害波动立刻切换 |
| **挑战怒吼 Challenging Shout** | 敌方全体接下来 2 回合强制攻击施放者 | 对所有怪物施加上述仇恨修正 |

- **强制攻击期间**：怪物无视仇恨表，仅攻击指定目标；强制次数在怪物每次行动后减 1。

## 四、与策略系统的衔接

### 4.1 目标规则 `highest-threat`

当玩家配置 `targetRule: 'highest-threat'` 时：

| 含义 | 实现 |
|------|------|
| **敌方目标** | 在存活怪物中，选择「对己方仇恨最高者不是自己的怪物」中，该最高仇恨值最大的那只；若无则选随机存活怪物 |
| **简化实现** | 选择「当前未在攻击自己的怪物」中，对己方总仇恨最高的那只；若全在打自己则随机 |

- **设计意图**：坦克优先攻击可能 OT 的目标，拉稳仇恨。

### 4.2 目标规则（新版仇恨二级，需指定坦克）

战术 UI 中「仇恨」下四项对应下列 `targetRule`（实现见 `pickTargetByRule`）：

| 规则 ID | 含义（简述） |
|---------|----------------|
| `threat-not-tank-random` | 仇恨表第一名**不是**指定坦克的怪中随机；若已存在非零威胁且全体第一仇恨均为坦克则**无目标**（开局全 0 威胁时仍在全体存活怪中随机）。详见 [10-tactics.md](10-tactics.md) |
| `threat-tank-top-random` | 仇恨表第一名**是**指定坦克的怪中随机 |
| `threat-tank-top-lowest-on-tank` | 在上一行同一子集内，对坦克仇恨**最低** |
| `threat-tank-top-highest-on-tank` | 在上一行同一子集内，对坦克仇恨**最高** |

「仇恨表第一名」指该怪对**存活英雄**威胁值最大的英雄 id。

### 4.3 兼容：`lowest-threat`（旧）

当玩家配置 `targetRule: 'lowest-threat'` 时：

| 含义 | 实现 |
|------|------|
| **敌方目标** | 在存活怪物中，选择「对施法者仇恨最低」的那只 |
| **实现** | 遍历存活怪物，取 `threat[monsterId][actorId]` 最小的怪物；并列时取第一个 |

- **设计意图**（旧版）：对施法者仇恨最低；与新版「坦克子集 + 对坦克最低」不同，旧存档仍按旧逻辑解析。

### 4.4 条件 `ally-ot`

| 条件 | 含义 | 实现 |
|------|------|------|
| **ally-ot** | 存在至少一个怪物，其仇恨最高目标不是指定坦克 | 遍历所有怪物，若任一只的「仇恨最高英雄」≠ 指定坦克，则 `ally-ot = true` |

- **坦克**：按 2.3 定义，为玩家指定的坦克；若未指定，则 `ally-ot` 恒为 false，且战术面板中该选项不可选。
- **战术 UI**：条件类别为「敌方」，条目为「OT」（内部键名 `ally-ot`；语义是每只怪的仇恨比较）。

### 4.5 友方目标 `tank`

治疗、Buff 等选择 `targetRule: 'tank'` 时，目标为 2.3 定义的**指定坦克**。未指定坦克时该选项不可选。

## 五、战斗流程中的仇恨结算

### 5.1 回合内顺序

1. **出手顺序**：按敏捷从高到低，英雄与怪物交错行动。
2. **英雄行动时**：造成伤害/治疗 → 立即更新对应怪物的仇恨表。
3. **怪物行动时**：先按 2.2 选择目标，再执行攻击；攻击后不改变仇恨（受击方仇恨由英雄造成伤害时产生，怪物普攻不给自己加仇恨）。

### 5.2 仇恨更新时机

| 时机 | 更新内容 |
|------|----------|
| 英雄对怪物造成伤害 | 该怪物对施放者的仇恨 += 伤害 × 技能仇恨倍率 |
| 英雄治疗 | 所有存活怪物对治疗者的仇恨 += 治疗量 × 0.5 |
| 英雄套盾（真言术：盾等） | 所有存活怪物对施放者的仇恨 += 吸收量 × 0.25（低仇恨） |
| 嘲讽/挑战怒吼 | 目标怪物对施放者的仇恨 = max(当前最高, 施放者当前仇恨) × 1.1 |

## 六、MVP 范围与后续扩展

### 6.1 MVP（基础版）

| 项目 | 说明 |
|------|------|
| 每怪独立仇恨表 | 初始 0，仅增不减 |
| 伤害 → 仇恨 | 1 伤害 = 1 仇恨（可配置技能倍率） |
| 治疗 → 仇恨 | 0.5 倍率，对所有怪物 |
| 套盾 → 仇恨 | 0.25 倍率（低仇恨），对所有怪物 |
| 嘲讽 | 强制 2 次行动 + 仇恨修正 |
| 怪物目标选择 | 强制 > 仇恨最高 > 随机 |
| ally-ot、新版仇恨四项、`highest-threat`/`lowest-threat`（兼容）、tank | 接入策略系统 |

### 6.2 仇恨展示 (Threat Display in Combat Log & UI)

为便于玩家复盘与战术设计，战斗日志与 UI 需体现仇恨变化。采用分层设计：

#### 6.2.1 第一层：关键时刻与目标原因

| 展示项 | 位置 | 格式/说明 |
|--------|------|-----------|
| **OT 事件** | 战斗日志 | 怪物切换目标时新增一条：`[R4] Wolf switched target to Mage (OT!)`；仅当目标切换到**非坦克**英雄时触发；怪物攻击坦克（战士）不显示 OT |
| **怪物攻击目标原因** | 战斗日志详情（log-detail-box） | 怪物攻击时在详情中显示：`Attacking Tank (highest threat)` 或 `Attacking Tank (taunted)` |
| **嘲讽效果说明** | 战斗日志详情（log-calc，与伤害公式同级） | Taunt 施放条目在详情中显示：`持续(2 次行动): 目标名 -> 施放者名`；名称着色与主行一致（目标 `monsterTierColor` / `classColor(targetClass)`，施放者 `classColor(actorClass)`） |

#### 6.2.2 第二层：仇恨数值与卡片指示

| 展示项 | 位置 | 格式/说明 |
|--------|------|-----------|
| **伤害/治疗附带仇恨** | 战斗日志详情 | 伤害条目详情中增加：`Threat +15 to Wolf`；治疗条目：`Threat +8 to all monsters` |
| **怪物卡片当前目标** | 怪物卡片 | 在怪物名称或 HP 旁显示 `→ Tank` 或 `→ Mage`，表示当前攻击目标 |

#### 6.2.3 实现顺序

1. 第一层：OT 事件、怪物攻击目标原因、Taunt 效果说明
2. 第二层：伤害/治疗仇恨提示、怪物卡片目标指示

### 6.3 后续可扩展

| 项目 | 说明 |
|------|------|
| 仇恨衰减 | 每回合或每 N 回合，未受攻击的怪物仇恨按比例衰减 |
| 减仇恨技能 | 如盗贼消失、法师冰箱等 |
| 怪物类型差异 | 部分怪物仇恨敏感度不同（如易被治疗吸引） |
| 仇恨明细面板 | 可展开的仇恨矩阵（每怪 × 每英雄），供深度分析 |
