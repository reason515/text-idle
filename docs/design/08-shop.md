# 商店系统设计详细说明

## 一、设计思路概述

商店借鉴《暗黑破坏神2》的**赌博 (Gambling)** 功能：玩家花费金币购买各种「未鉴定」的装备，购买后自动执行鉴定，可能获得该装备的普通、魔法、稀有乃至暗金等各成色版本。核心乐趣在于「花钱开盲盒」的期待感与惊喜感，为金币提供稳定的消耗途径，与战斗掉落、出售装备形成经济闭环。

## 二、购买与鉴定流程

| 阶段 | 说明 |
|------|------|
| **选择部位** | 玩家在商店界面选择要购买的装备部位；武器与副手按类型细分（见「商店可购买槽位」） |
| **支付金币** | 扣除对应部位的价格，金币不足时无法购买 |
| **生成未鉴定装备** | 系统按规则生成一件「未鉴定」装备（部位、底材、等级已确定，品质与词缀未确定） |
| **自动鉴定** | 购买后**立即自动执行鉴定**，复用与怪物掉落相同的品质与词缀随机逻辑 |
| **进入背包** | 鉴定后的装备直接进入玩家背包；若背包已满则按背包满载规则处理（丢弃并提示） |

- **设计说明**：购买即鉴定，无需玩家手动鉴定。鉴定逻辑与怪物掉落完全一致，保证规则透明、实现可复用。

## 三、装备等级与层级

| 规则 | 说明 |
|------|------|
| **等级上限** | 商店出售的装备等级 **不超过玩家小队角色的最高等级**；若小队为空，则默认等级为 1 |
| **层级与底材** | 根据装备等级，按「装备层级与怪物掉落」规则确定可选的层级（普通/拓展/精英）与底材族；与怪物掉落逻辑一致 |
| **等级基准** | 与[02-levels-monsters.md](./02-levels-monsters.md) 2.6 等级相关机制统一规则一致：以队伍中最高英雄等级为基准 (baseLevel) |

- **设计意图**：避免玩家通过商店获得远超当前进度的装备，保持与战斗掉落的节奏一致；同时保证商店装备始终「可用」。

## 四、品质与词缀（复用掉落逻辑）

商店装备的品质（普通/魔法/稀有/暗金）与词缀随机，复用掉落逻辑，但**成色分布更偏向蓝/黄**以奖励付费购买：

| 维度 | 说明 |
|------|------|
| **寻宝值 (MF)** | 商店鉴定时，可使用玩家队伍的 MF 参与成色 roll（可配置；若无 MF 则按基准概率） |
| **成色分布** | 商店专用：普通 50%、魔法 35%、稀有 15%（高于普通怪掉落的 92%/7%/1%，与 Boss 相当） |
| **词缀** | 魔法/稀有装备从词缀池随机抽取；暗金装备使用固定属性与特效 |
| **戒指/项链** | 戒指、项链仅产出蓝色及以上成色，与掉落规则一致 |

- **可配置**：商店是否受 MF 影响、成色概率是否与掉落完全一致或略有调整，均可通过配置调节。

## 五、价格设计

| 规则 | 说明 |
|------|------|
| **按部位定价** | 不同装备部位有不同基础价格；可与出售价格公式中的「部位系数」挂钩，保持一致性 |
| **等级影响** | 装备等级越高，价格越高；公式：**价格 = 基础价 × (1 + 等级 × 0.08)**，等级系数 0.08 |
| **透明化** | 购买前显示该部位、该等级区间的价格范围或具体价格，避免「盲买」 |

- **平衡目标**：价格应使商店成为「有成本的选择」，既消耗金币又不过于廉价，与战斗掉落形成互补而非替代。
- **示例**：Helm 基础价 250，1 级约 270 金，10 级约 450 金；2H 武器基础价 600，1 级约 648 金。

## 六、商店可购买槽位

武器与副手按类型细分，便于玩家定向购买：

| 分类 | 槽位 | 底材类型 | 说明 |
|------|------|----------|------|
| **武器** | 1H Weapon (Phys) | MainHand | 匕首、剑、斧、锤等单手物理主手 |
| | 2H Weapon (Phys) | MainHand2H | 双手剑（Claymore、Great Sword 等） |
| | 2H Weapon (Bow) | MainHand2HBow | 双手弓（Short Bow、Composite Bow 等） |
| | 2H Weapon (Magic) | MainHand2HStaff | 双手杖（Short Staff、Sacred Staff 等） |
| | 1H Weapon (Magic) | MainHandWand | 权杖 / 魔杖（Scepter、Wand 等） |
| **副手** | Shield | Shield | 盾牌（物理防护） |
| | Orb | OffHand | 法术宝珠（法系副手） |
| **防具** | Helm, Body Armor, Gloves, Boots, Belt | 对应底材 | 头盔、盔甲、手套、鞋子、腰带 |
| **饰品** | Amulet, Ring | 对应底材 | 项链、戒指 |

- **底材扩展**：`itemBases.js` 中需包含 MainHand2H（双手剑）、MainHand2HBow（双手弓）、MainHand2HStaff（双手杖）、MainHandWand（权杖/魔杖）底材表，与 MainHand（单手物理）并列。

## 七、商店 UI 入口

| 规则 | 说明 |
|------|------|
| **入口位置** | 主界面**顶部栏**，在**背包按钮旁边**增加「商店」按钮/图标 |
| **交互方式** | 点击后以 Modal 弹窗展示商店界面 |
| **商店界面** | 按分组展示：Weapons / Armor / Accessories；每组内按槽位列出，显示价格与当前金币；金币数字使用 `var(--color-gold)`；分组标题使用 `var(--accent)`；Modal 设置 `max-height: 85vh` 支持滚动；点击购买后扣款并执行鉴定，装备进入背包 |

## 八、与核心循环的衔接

| 循环阶段 | 商店系统贡献 |
|----------|--------------|
| **放置 (Engage)** | 战斗与出售获得金币，为商店消费提供资源 |
| **优化 (Optimize)** | 通过商店购买补充特定部位的装备缺口，或追求高品质 roll |
| **经济 (Economy)** | 金币的重要消耗途径，形成「战斗/出售 → 金币 → 商店赌博 → 装备」的闭环 |
