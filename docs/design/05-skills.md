
假设玩家将全部 295 点自由属性投入主属性（极端主属性构筑），则 60 级时主属性 ≈ 1级基础 + 295。示例（战士，全加力量）：

| 职业 | 1级主属性 | 60级主属性（全加主属性） | HP 估算 | PhysAtk/SpellPower 估算 |
|------|-----------|--------------------------|---------|-------------------------|
| 战士 | Str 10 | 10 + 295 = 305 | 10 + 9×4 + 60×2 = 166 | 5 + 305×0.65 ≈ 203 |
| 法师 | Int 11 | 11 + 295 = 306 | 10 + 4×2 + 60×2 = 138 | 5 + 306×0.65 ≈ 204 |
| 盗贼 | Agi 11 | 11 + 295 = 306 | 10 + 6×2.8 + 60×2 ≈ 147 | 5 + 306×0.55 ≈ 173 |

- **均衡分配**：若玩家将 295 点分散至多属性（如坦克加耐力、DPS 加主属性），实际数值介于 1 级与上述极端值之间。装备词缀会进一步放大差异。

#### 2.2.3 伤害与减伤公式

| 类型 | 公式 | 说明 |
|------|------|------|
| 物理原始伤害 | `rawDamage = baseRoll * physMultiplier * SkillCoeff * [1.5 if crit] + physAtkBonus` | 见 2.2.3.1 |
| 法术原始伤害 | `rawDamage = baseRoll * spellMultiplier * SkillCoeff * [1.5 if crit] + spellPowerBonus` | 见 2.2.3.2 |
| 护甲抵消 | `finalDamage = max(1, rawDamage - Armor)` | 1 护甲 = 抵消 1 点物理伤害，无上限，装备可叠加 |
| 抗性抵消 | `finalDamage = max(1, rawDamage - Resistance)` | 1 抗性 = 抵消 1 点魔法伤害 |
| 暴击伤害 | `CritMultiplier = 1.5` | 暴击时伤害 * 1.5，可由装备/技能修改 |

- **直接抵消**：每点护甲/抗性 = 抵消 1 点伤害，无上限，装备可叠加。

#### 2.2.3.1 物理伤害公式（空手 + 武器统一结构）

**设计意图**：空手与武器使用同一套结构，空手基础伤害为范围 1–4，武器伤害叠加到该范围；主属性采用乘法加成，增加随机性与上下限差异。

| 概念 | 说明 |
|------|------|
| 空手基础 | 每次攻击 roll `unarmedRoll = random(1, 4)` |
| 武器贡献 | 有武器时 `weaponRoll = random(weaponMin, weaponMax)`；无武器时为 0 |
| 基础骰子 | `baseRoll = unarmedRoll + weaponRoll`；空手 1–4，有武器 3–5 时 4–9 |
| 主属性乘数 | `physMultiplier = 1 + baseAttr * 0.20`；力量职业 baseAttr = Str×1.4 + Agi×0.6，敏捷职业 baseAttr = Agi×1.4 + Str×0.6 |
| 非武器加成 | 词缀、戒指等提供的 +PhysAtk 为固定值 `physAtkBonus` |
| 法术武器 | 魔杖/权杖的 SpellPower+ 同理可为范围，每次法术伤害时随机 |

**公式**（物理伤害）：

```
baseRoll = random(1, 4) + [有武器 ? random(weaponMin, weaponMax) : 0]
physMultiplier = 1 + baseAttr * 0.20
rawDamage = round(baseRoll * physMultiplier) + physAtkBonus
finalDamage = max(1, rawDamage * SkillCoeff * [1.5 if crit] - targetArmor)
```

- `baseAttr`：按职业主属性计算（力量职业 Str×1.4 + Agi×0.6，敏捷职业 Agi×1.4 + Str×0.6）
- `weapon.physAtkMin/Max`：该武器实例掉落时 roll 出的上下限；无武器时 baseRoll 仅 1–4
- `physAtkBonus`：非武器装备（词缀、戒指等）提供的固定 PhysAtk

**UI 与透明化**：

- 武器 tooltip 显示该实例 roll 后的「PhysAtk: 3–5」或「伤害: 3–5」（上下限均为掉落时随机结果）
- 若有底材参考，可显示「底材范围: 2–4」（表示同类武器的可能 roll 范围）
- 角色面板 PhysAtk 可显示为「17–39」（baseRoll 范围 × physMultiplier + physAtkBonus）
- 战斗日志中每次伤害为实际 roll 后的数值，便于复盘

#### 2.2.3.2 法术伤害公式（与物理统一结构）

**设计意图**：法术伤害与物理伤害采用同一套结构，空手基础 1–4，法杖贡献叠加；主属性乘法加成。

| 概念 | 说明 |
|------|------|
| 空手基础 | 每次施法 roll `unarmedRoll = random(1, 4)` |
| 法杖贡献 | 有法杖时 `weaponRoll = random(weaponMin, weaponMax)`；无武器时为 0 |
| 基础骰子 | `baseRoll = unarmedRoll + weaponRoll` |
| 主属性乘数 | `spellMultiplier = 1 + baseAttr * 0.20`；baseAttr = Int×1.2 + Spirit×0.8 |
| 非武器加成 | 词缀、戒指等提供的 +SpellPower 为固定值 `spellPowerBonus` |

**公式**（法术伤害）：

```
baseRoll = random(1, 4) + [有法杖 ? random(weaponMin, weaponMax) : 0]
spellMultiplier = 1 + baseAttr * 0.20
rawDamage = round(baseRoll * spellMultiplier) + spellPowerBonus
finalDamage = max(1, rawDamage * SkillCoeff * [1.5 if crit] - targetResistance)
```

- `baseAttr`：Int×1.2 + Spirit×0.8
- `weapon.spellPowerMin/Max`：法杖实例掉落时 roll 出的上下限
- `spellPowerBonus`：非武器装备提供的固定 SpellPower

#### 2.2.4 怒气与能量规则（草案）

| 资源 | 获得方式 | 消耗示例 |
|------|----------|----------|
| 怒气 | 受到伤害: +1 怒气/2 点伤害（至少 1 点）；造成伤害: +1 怒气/4 点伤害 | 英勇打击 15 怒气，盾牌猛击 20 怒气 |
| 能量 | 固定每回合恢复 10 点 | 影袭 40 能量，背刺 60 能量 |
| 集中值 | 固定每回合恢复 6 点；稳固射击不消耗 | 奥术射击 30 集中值 |

- 具体数值在技能设计时平衡，此处为量级参考。

### 2.3 属性来源

| 来源 | 说明 |
|------|------|
| 基础属性 | 职业与等级决定的基础值 |
| 升级成长 | 每级获得固定点数，按职业主属性分配 |
| 装备词缀 | 装备提供 +Str、+Agi、+Int、+护甲、+法术抗性 等词缀 |
| Buff/Debuff | 技能或状态效果临时增减属性 |

### 2.4 属性显示与透明化

- 角色面板展示：**基础属性** + **装备加成** + **Buff 加成** = **最终属性**。
- **PhysAtk/SpellPower 公式悬停**：鼠标悬停时展示完整公式，含 baseAttr 详细计算（物理：Str×1.4+Agi×0.6 或 Agi×1.4+Str×0.6；法术：Int×1.2+Spi×0.8），便于玩家理解伤害来源。
- 战斗日志与数据统计中，可展示关键衍生数值（如实际暴击率、命中率），便于玩家验证与优化构筑。
- **Buff/Debuff 面板提示**：角色或怪物身上存在 buff/debuff 时，在其面板上显示相应徽章；鼠标悬停可查看详细说明（名称、剩余回合、效果描述）。

# 技能系统设计详细说明

## 一、设计思路概述

技能系统从《魔兽世界》经典职业技能中汲取灵感，经适配与改造后服务于回合制战斗。核心设计围绕**技能来源与适配**、**初始选择**、**成长节奏**、**释放控制**四个维度展开。

## 二、技能来源与适配

### 2.1 选型原则

| 原则 | 说明 |
|------|------|
| **来源** | 从《魔兽世界》经典职业技能中选择适合回合制游戏的技能 |
| **适配** | 对不适合的技能进行改造：删除实时单位（秒、分钟），改为回合制单位（回合、每回合） |
| **职业绑定** | 每个职业拥有独立的技能池，技能为职业专属 |

### 2.2 适配示例

| 原 WoW 设计 | 改造后（回合制） |
|------------|------------------|
| 3 秒冷却 | 3 回合冷却 |
| 每秒恢复 10 法力 | 每回合恢复 10 法力 |
| 持续 12 秒的 Buff | 持续 12 回合的 Buff |

- **设计意图**：保持 WoW 技能风味，同时严格遵循「回合制」设计哲学（见 game-design.mdc）。

## 三、初始技能选择

### 3.1 触发时机

| 规则 | 说明 |
|------|------|
| **触发** | 新角色加入队伍时（招募时） |
| **选择数量** | 从本职业的 **3 个初始技能** 中选择 **1 个** |
| **结果** | 所选技能成为该角色首个可用技能，可立即在战斗中使用 |

**扩展英雄**：若新英雄以 5 级（或 5 的倍数）加入（见 [02-levels-monsters.md](./02-levels-monsters.md) 1.2.1），招募流程中需先完成初始技能选择，再完成该等级的技能选择（加强已有或学习新技能），二者均在加入队伍前完成。

### 3.2 设计意图

- 让玩家在招募阶段即参与构筑决策，塑造角色定位（如坦克选仇恨技能、治疗选治疗技能）。
- 3 选 1 提供差异化，同职业不同角色可走不同路线。

## 四、技能成长（等级解锁）

### 4.1 触发等级

| 规则 | 说明 |
|------|------|
| **触发等级** | 角色等级为 **5 的倍数** 时（5、10、15、20、25…60） |
| **选择窗口** | 弹出技能选择界面，玩家可选择加强或学习新技能；若暂不选择，游戏继续进行（与升级属性加点类似） |

### 4.2 二选一机制

每次触发时，玩家在以下两种选项中**二选一**：

| 选项 | 说明 |
|------|------|
| **加强已有技能** | 提升该技能的数值或效果（如伤害倍率、治疗量、冷却缩短等） |
| **学习新技能** | 从技能池中学习一个尚未拥有的新技能 |

### 4.3 学习新技能时的可选池

| 规则 | 说明 |
|------|------|
| **可选数量** | 每个选择窗口提供 **3 个固定的可选新技能** |
| **三系分布** | 3 个选项分别来自职业的三个派系（如战士：武器、狂暴、防护），便于玩家按派系构筑 |
| **固定性** | 同一等级、同一职业的 3 个选项为固定组合（非随机），便于玩家规划与攻略 |
| **排除已学** | 已拥有的技能不会出现在可选列表中 |
| **池耗尽** | 若技能池中未学技能不足 3 个，则显示剩余全部；若已学满，则仅显示「加强已有技能」 |

### 4.4 成长节奏示例

| 等级 | 可选操作 |
|------|----------|
| 5 | 加强已有技能，或学习技能 A/B/C（3 选 1，分属三系） |
| 10 | 加强已有技能，或学习技能 D/E/F（3 选 1，分属三系） |
| … | … |
| 60 | 加强已有技能，或学习高级技能（3 选 1，分属三系） |

- **满级技能数**：60 级时共触发 12 次（5、10、15…60），理论上可拥有 1（初始）+ 12 = 最多 13 个技能，或通过多次「加强」深化少量技能。

## 五、技能释放控制

### 5.1 当前阶段（MVP）

| 规则 | 说明 |
|------|------|
| **释放方式** | 由 **AI 自动释放** |
| **AI 逻辑** | 采用简单规则：如按预设优先级选择技能、资源充足时释放、目标选择（仇恨最高/血量最低等） |
| **玩家影响** | 玩家通过「选择学哪些技能、加强哪些技能」间接影响战斗表现 |

### 5.2 后续扩展（策略配置）

| 规则 | 说明 |
|------|------|
| **策略配置** | 玩家可自定义技能使用策略 |
| **可配置项** | 技能释放优先级、目标选择规则（仇恨最高/血量最低/随机）、触发条件（如 HP 低于 X% 时治疗） |
| **与核心循环衔接** | 对应「布局 (Strategize)」：为每位角色配置战斗 AI，使策略自动执行 |

- **设计意图**：先实现简单 AI 保证可玩性，后续开放策略配置以深化「布局」乐趣。

## 六、技能分类与结构（预留）

| 类型 | 说明 | 备注 |
|------|------|------|
| 主动技能 | 消耗资源、需选择目标、有冷却 | 主要设计对象 |
| 被动技能 | 常驻效果，无需释放 | 后续扩展 |
| 普攻 | 无消耗或低消耗的基础攻击 | 每回合默认可用 |

- 具体技能列表、数值公式、冷却与消耗等，见下方「技能池设计」章节。

## 七、与核心循环的衔接

| 循环阶段 | 技能系统贡献 |
|----------|--------------|
| **组建 (Assemble)** | 招募时选择初始技能；等级成长时选择加强或新技能 |
| **布局 (Strategize)** | 当前为 AI 自动释放；后续为玩家配置技能策略 |
| **放置 (Engage)** | 技能在战斗中自动执行，影响伤害、治疗、仇恨等 |
| **分析 (Analyze)** | 战斗日志与数据统计展示技能使用频率、伤害占比等 |
| **优化 (Optimize)** | 根据数据调整技能搭配、加强优先级、策略配置 |

## 八、技能池设计（职业）

### 8.1 战士 (Warrior)

#### 8.1.1 怒气机制 (Rage)

| 规则 | 说明 |
|------|------|
| **初始怒气** | 战斗开始时怒气为 0 |
| **怒气上限** | 固定 100，不随等级变化 |
| **怒气获得** | 每次发起攻击或受到攻击（命中时）增加固定怒气；暴击时怒气翻倍；闪避不增加怒气 |
| **怒气消耗** | 使用消耗怒气的技能时扣除；怒气不足时无法使用该技能 |
| **脱战** | 战斗结束后怒气归零 |
| **休息** | 休息阶段不恢复怒气 |

- 伤害公式沿用[04-classes-attributes.md](./04-classes-attributes.md)2.2.3；SkillCoeff 为技能倍率。

#### 8.1.2 战士三派系（WoW 经典版对应）

| 派系 | 英文 | 定位 | 典型技能 |
|------|------|------|----------|
| 武器 | Arms | 单体输出、控制、战术 | 英勇打击、撕裂、雷霆一击、顺劈斩、致死打击、斩杀、断筋、横扫攻击、压制、猛击、战斗怒吼、反击风暴、冲锋 |
| 狂暴 | Fury | 多目标、爆发、续航 | 怒击、旋风斩、嗜血、狂怒、猛击、狂暴打击、血性狂怒、狂暴之怒、死亡之愿、怒气澎湃、泰坦之握 |
| 防护 | Protection | 坦克、仇恨、减伤 | 破甲（护甲超出时额外伤害）、嘲讽、盾牌猛击、复仇、盾牌格挡、震荡猛击、盾墙、挫志怒吼、破釜沉舟、挑战怒吼、盾牌壁垒 |

- **战斗怒吼**：WoW 中为全职业基础技能，本作归入**武器**系（战术指挥、团队增益）。
- **反击风暴**：WoW 经典版为**武器**系天赋，非狂暴。
- 每个技能选择窗口的 3 个选项分别来自上述三派系；等级越高，技能越强。

#### 8.1.3 初始技能（招募时 3 选 1，分属三系，最基础）

初始仅 1 个技能，需在单人场景下也有价值；三者定位区分清晰、相对平衡。

| 派系 | 技能 | 英文 | 消耗 | 冷却 | 效果 | 加强效果 |
|------|------|------|------|------|------|----------|
| 武器 | 英勇打击 | Heroic Strike | 15 怒气 | 0 | 对单体造成物理伤害，倍率 1.2；武器系标准输出 | 倍率 +0.2/次（最高 1.8，最多 3 次） |
| 狂暴 | 嗜血 | Bloodthirst | 20 怒气 | 0 | 对单体造成物理伤害，倍率 1.2，并恢复造成伤害 15% 的生命值；狂暴系续航输出，与武器纯伤害区分 | 倍率 +0.1、吸血 +5%/次（最多 3 次） |
| 防护 | 破甲 | Sunder Armor | 15 怒气 | 0 | 对单体造成物理伤害（倍率 0.8），并使目标护甲 -8，持续 3 回合；若护甲削减后低于 0，每超出 1 点本次伤害 +2%；可叠加层数；高仇恨 | 可叠加层数：每增强 1 次最大层数 +1（最高 4 层），每次施放叠加 1 层并刷新持续时间 |

- **嗜血**：武器与狂暴初始均为纯伤害时过于相似；改为**嗜血**，突出狂暴系续航（吸血），与武器系纯输出区分。
- **破甲**：倍率固定 0.8；若护甲削减后低于 0，每超出 1 点本次伤害 +2%；可叠加层数并刷新持续时间。

#### 8.1.4 等级解锁技能（5 的倍数时，加强已有 或 3 选 1 学新，分属三系，随等级递进增强）

**等级 5（入门）**

| 武器 | 顺劈斩 | Cleave | 20 怒气 | 0 | 对 2 个目标造成物理伤害（倍率 0.7）；WoW 武器系，多目标直接伤害 | 目标数 +1（最高 4）；或倍率 +0.1 |
| 狂暴 | 旋风斩 | Whirlwind | 25 怒气 | 2 回合 | 对敌方全体造成物理伤害（倍率 0.55）；狂暴系唯一全体 AOE | 倍率 +0.05/次（最高 0.8）；或冷却 -1 回合 |

#### 8.1.6 战士初始技能增强（实现规格）

玩家在 5 级、10 级等技能选择窗口选择「强化已有技能」时，`hero.skillEnhancements[skillId].enhanceCount` 递增。**每个技能最多强化 3 次**（enhanceCount 上限 3）。战斗执行技能时，需根据 `enhanceCount` 计算**增强后的技能参数**，再参与伤害/治疗/debuff 计算。以下为战士三个初始技能的增强公式（实现时按 enhanceCount 应用）。

| 技能 ID | 技能名 | 基础参数 | 增强类型 | 公式 | 上限 |
|---------|--------|----------|----------|------|------|
| heroic-strike | 英勇打击 | coefficient: 1.2 | 倍率 | coefficient = 1.2 + enhanceCount × 0.2 | 1.8 |
| bloodthirst | 嗜血 | coefficient: 1.2, healPercent: 0.15 | 倍率 + 吸血 | coefficient = 1.2 + enhanceCount × 0.1；healPercent = 0.15 + enhanceCount × 0.05 | 1.5；0.30 |
| sunder-armor | 破甲 | coefficient: 0.8, perStackArmorReduction: 8, excessDamagePercent: 2, debuffDuration: 3 | 可叠加层数 | maxStacks = 1 + enhanceCount；每次施放叠加 1 层并刷新持续时间 | 4 层 |

**说明**：

- **英勇打击**：每次增强 +0.2 倍率，最高 1.8；enhanceCount = 3 时达到上限（1.2 + 0.6 = 1.8）。
- **嗜血**：每次增强同时提升伤害与回血：倍率 +0.1、吸血 +5%；3 次增强后 coefficient = 1.5，healPercent = 0.30（30%）。倍率增幅低于英勇打击，因另有吸血增益。
- **破甲**：倍率固定 0.8；若护甲削减后低于 0，每超出 1 点本次伤害 +2%。增强后支持**叠加层数**：基础 1 层（每层 -8 护甲）；每增强 1 次，最大层数 +1（最高 4 层）。每次施放破甲时：目标已有破甲 debuff 则叠加 1 层（不超过 maxStacks），并**刷新持续时间**至 3 回合；无 debuff 则施加 1 层。总护甲削减 = 8 × 当前层数。超出值 = max(0, 本次削减量 - 削减前护甲)。

**实现要点**：

- 战斗前或执行技能时，根据 `hero.skillEnhancements[skillId]?.enhanceCount ?? 0` 计算增强后的技能参数；enhanceCount 超过 3 时按 3 处理。
- 破甲 debuff 结构需支持 `stacks` 字段；`armorReduction = perStackArmorReduction × stacks`。
- 技能描述（effectDesc）在 UI 中应反映增强后的数值，便于玩家感知。

**等级解锁技能的增强**：5 级及以上的顺劈斩、旋风斩、嘲讽等，其增强公式在 8.1.4 各等级表格的「加强效果」列中已有描述，实现时按相同模式扩展；**同样遵循每技能最多强化 3 次**。本小节仅规范战士初始三个技能的增强。

### 8.2 法师 (Mage)

#### 8.2.1 法力机制 (Mana)

| 规则 | 说明 |
|------|------|
| **初始法力** | 战斗开始时法力为满值 |

| **伤害类型** | 法术伤害，受目标**抗性 (Resistance)** 抵消 |

#### 8.2.6 法师初始技能增强（实现规格）

玩家在 5 级、10 级等技能选择窗口选择「强化已有技能」时，`hero.skillEnhancements[skillId].enhanceCount` 递增。**每个技能最多强化 3 次**（enhanceCount 上限 3）。战斗执行技能时，需根据 `enhanceCount` 计算**增强后的技能参数**，再参与伤害/debuff 计算。以下为法师三个初始技能的增强公式（实现时按 enhanceCount 应用）。

| 技能 ID | 技能名 | 基础参数 | 增强类型 | 公式 | 上限 |
|---------|--------|----------|----------|------|------|
| arcane-blast | 奥术冲击 | coefficient: 1.2 | 倍率 | coefficient = 1.2 + enhanceCount × 0.2 | 1.8 |
| fireball | 火球术 | coefficient: 1.2, burnCoeff: 0.05, burnDuration: 3 | 倍率 + 灼烧 | coefficient = 1.2 + enhanceCount × 0.1；burnCoeff = 0.05 + enhanceCount × 0.02；burnDuration 可 +1 回合 | 1.5；0.11；4 回合 |
| frostbolt | 寒冰箭 | baseCoefficient: 1.0, refreshCoefficient: 1.2, perStackResistReduction: 6, debuffDuration: 3 | 可叠加层数 | maxStacks = 1 + enhanceCount；每次施放叠加 1 层并刷新持续时间 | 4 层 |

**说明**：

- **奥术冲击**：每次增强 +0.2 倍率，最高 1.8；enhanceCount = 3 时达到上限。
- **火球术**：每次增强同时提升伤害与灼烧：倍率 +0.1、灼烧系数 +0.02、灼烧持续可 +1 回合；3 次增强后 coefficient = 1.5，burnCoeff = 0.11。
- **寒冰箭**：增强后支持**叠加层数**。基础 1 层（每层 -6 抗性）；每增强 1 次，最大层数 +1（最高 4 层）。每次施放寒冰箭时：目标已有 debuff 则叠加 1 层（不超过 maxStacks），并**刷新持续时间**至 3 回合；无 debuff 则施加 1 层。总抗性削减 = 6 × 当前层数。

**实现要点**：

- 战斗前或执行技能时，根据 `hero.skillEnhancements[skillId]?.enhanceCount ?? 0` 计算增强后的技能参数；enhanceCount 超过 3 时按 3 处理。
- 寒冰箭 debuff 结构需支持 `stacks` 字段；`resistanceReduction = perStackResistReduction × stacks`。
- 灼烧 debuff 结构需支持 `burnCoeff`、`remainingRounds`；每回合伤害 = SpellPower × burnCoeff。
- 技能描述（effectDesc）在 UI 中应反映增强后的数值，便于玩家感知。

**等级解锁技能的增强**：5 级及以上的奥术飞弹、冰霜新星、烈焰风暴等，其增强公式在各等级表格的「加强效果」列中已有描述，实现时按相同模式扩展；**同样遵循每技能最多强化 3 次**。本小节仅规范法师初始三个技能的增强。
